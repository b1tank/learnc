> Modify the pattern-finding program (from §4.4 / §5.5) so it reads from one or more files named on the command line, prefixing each output line with the file name. If no files are given, read `stdin`.

## Approach

A small wrapper around the line-grep:

```c
for each filename in argv[1..]:
    fp = fopen(filename, "r")
    grep_file(fp, filename, pattern)
    fclose(fp)
```

If no files, treat `stdin` as a single "file" with name `""` (don't prefix lines).

```c:starter
#include <stdio.h>
#include <string.h>

static int grep_file(FILE *fp, const char *name, const char *pattern) {
    char line[4096];
    int found = 0, lineno = 0;
    while (fgets(line, sizeof line, fp)) {
        ++lineno;
        if (strstr(line, pattern)) {
            if (name) printf("%s:%d: %s", name, lineno, line);
            else      printf("%d: %s",         lineno, line);
            ++found;
        }
    }
    return found;
}

int main(int argc, char *argv[]) {
    if (argc < 2) {
        fprintf(stderr, "usage: %s PATTERN [FILE...]\n", argv[0]);
        return 2;
    }
    const char *pattern = argv[1];

    if (argc == 2) {
        return grep_file(stdin, NULL, pattern) ? 0 : 1;
    }

    int total = 0;
    /* if multiple files, always prefix with the filename */
    const char *prefix = (argc > 3) ? "yes" : NULL;
    for (int i = 2; i < argc; ++i) {
        FILE *fp = fopen(argv[i], "r");
        if (!fp) { perror(argv[i]); continue; }
        total += grep_file(fp, (argc > 3) ? argv[i] : NULL, pattern);
        fclose(fp);
    }
    return total ? 0 : 1;
}
```

## Why prefix only with multiple files?

The Unix `grep` does this by default - single file means no prefix (the prefix would be redundant), multiple files means each line gets its filename. The behaviour matches `grep -H`/`-h`.

## Try it

1. Add `-n` to force line numbers and `-H` to force filename prefix.
2. Add `-v` for "lines NOT matching".
3. Add `-i` for case-insensitive matching (lowercased compare).

## Notes from the author

- This is the skeleton of `grep(1)` - minus the regex engine. K&R Ch 4 builds a primitive matcher; real `grep` uses a deterministic finite automaton derived from the regex.
- Returning 0 only when something was found is the `grep` convention. Scripts use `if grep -q pattern file; then ...` to test for presence.
- For files that don't exist or aren't readable, the right behaviour is to print a warning to `stderr` and continue with the rest - don't bail. Most Unix tools follow this rule.
