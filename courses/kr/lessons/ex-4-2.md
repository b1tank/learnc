> Extend the `atof` from §4.2 so it handles scientific notation like `123.45e-6` or `2E10` (an optional `e` or `E` followed by an optionally-signed integer).

## Approach

After parsing the mantissa (sign, integer part, fraction), check for `e`/`E`. If found, parse an optional sign and a sequence of digits into an integer exponent. Then multiply or divide the result by `10^|exp|`.

```c:starter
#include <stdio.h>
#include <ctype.h>

double atof_x(const char s[]);

int main(void) {
    printf("'2.5'        -> %g\n", atof_x("2.5"));
    printf("'1e3'        -> %g\n", atof_x("1e3"));
    printf("'-3.14e2'    -> %g\n", atof_x("-3.14e2"));
    printf("'6.022e23'   -> %g\n", atof_x("6.022e23"));
    printf("'1.5E-2'     -> %g\n", atof_x("1.5E-2"));
    printf("' +0.5e+1'   -> %g\n", atof_x("  +0.5e+1"));
    return 0;
}
```

## Solution

```c
double atof_x(const char s[]) {
    double val   = 0.0;
    double power = 1.0;
    int    i     = 0;
    int    sign  = 1;

    while (isspace((unsigned char)s[i]))
        ++i;
    if (s[i] == '+' || s[i] == '-')
        sign = (s[i++] == '+') ? 1 : -1;

    for (; isdigit((unsigned char)s[i]); ++i)
        val = 10.0 * val + (s[i] - '0');
    if (s[i] == '.')
        ++i;
    for (; isdigit((unsigned char)s[i]); ++i) {
        val = 10.0 * val + (s[i] - '0');
        power *= 10.0;
    }
    val = sign * val / power;

    /* exponent */
    if (s[i] == 'e' || s[i] == 'E') {
        ++i;
        int exp_sign = 1;
        if (s[i] == '+' || s[i] == '-')
            exp_sign = (s[i++] == '+') ? 1 : -1;
        int exp = 0;
        for (; isdigit((unsigned char)s[i]); ++i)
            exp = 10 * exp + (s[i] - '0');
        /* apply 10^(exp_sign * exp) by repeated multiplication */
        double scale = 1.0;
        for (int k = 0; k < exp; ++k)
            scale *= 10.0;
        if (exp_sign > 0) val *= scale;
        else              val /= scale;
    }
    return val;
}
```

```output
'2.5'        -> 2.5
'1e3'        -> 1000
'-3.14e2'    -> -314
'6.022e23'   -> 6.022e+23
'1.5E-2'     -> 0.015
' +0.5e+1'   -> 5
```

## Notes from the author

- A repeated-multiplication loop for `10^exp` is fine for small exponents. For large exponents you'd use `pow(10.0, exp)` (in `<math.h>`); the standard library's implementation is more accurate (it doesn't accumulate rounding error). For Avogadro's number you get away with it because `double` has ~15.95 decimal digits of precision.
- The standard `strtod` is the right production tool - it handles hex floats, infinity, NaN, locale, and tells you where parsing stopped. Re-implementing it is for the learning value.
- Notice how each stage of the parse (sign, integer, fraction, exponent) is a self-contained loop. That sequential structure is what makes hand-written parsers debuggable; jamming everything into a single while loop would be hard to follow.
