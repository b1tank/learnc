> Write a program that reads a C source file and reports all pairs of identifiers that differ only in capitalisation (e.g. `MAX` vs `Max` vs `max`). This is a common source of bugs in case-sensitive languages.

## Approach

For each identifier, compute its lower-cased form as a *key*. Group identifiers by key. After scanning, any key with more than one distinct original spelling is reported.

A hash table maps the lowercase key to a list of original spellings.

```c:starter
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <ctype.h>

#define HASHSIZE 257

struct alias {
    char *original;
    struct alias *next;
};

struct entry {
    char *key;            /* lower-cased */
    struct alias *aliases;
    struct entry *next;
};

static struct entry *table[HASHSIZE];

static unsigned hash(const char *s) {
    unsigned h = 0;
    while (*s) h = 31u * h + (unsigned char)*s++;
    return h % HASHSIZE;
}

static void record(const char *original) {
    char *key = strdup(original);
    for (char *p = key; *p; ++p) *p = tolower((unsigned char)*p);

    unsigned h = hash(key);
    struct entry *e;
    for (e = table[h]; e; e = e->next)
        if (strcmp(e->key, key) == 0) break;
    if (!e) {
        e = malloc(sizeof *e);
        e->key = key;
        e->aliases = NULL;
        e->next = table[h];
        table[h] = e;
    } else {
        free(key);
        /* check if this spelling already recorded */
        for (struct alias *a = e->aliases; a; a = a->next)
            if (strcmp(a->original, original) == 0)
                return;
    }
    struct alias *a = malloc(sizeof *a);
    a->original = strdup(original);
    a->next = e->aliases;
    e->aliases = a;
}

static int alias_count(struct alias *a) {
    int n = 0;
    for (; a; a = a->next) ++n;
    return n;
}

static int getword(char *w, int lim) {
    int c;
    while ((c = getchar()) != EOF && !isalpha(c) && c != '_')
        ;
    if (c == EOF) return EOF;
    char *p = w;
    *p++ = c;
    while (--lim > 0) {
        c = getchar();
        if (!isalnum(c) && c != '_') break;
        *p++ = c;
    }
    *p = '\0';
    return w[0];
}

int main(void) {
    char buf[128];
    while (getword(buf, sizeof buf) != EOF)
        record(buf);

    for (int i = 0; i < HASHSIZE; ++i)
        for (struct entry *e = table[i]; e; e = e->next)
            if (alias_count(e->aliases) > 1) {
                printf("clash on '%s': ", e->key);
                for (struct alias *a = e->aliases; a; a = a->next)
                    printf("%s%s", a->original, a->next ? ", " : "\n");
            }
    return 0;
}
```

```output
(awaits text on stdin)
```

Example: feed it `Max max MAX max\n`.

```output
clash on 'max': MAX, max, Max
```

## Why this matters

Some C codebases mix `static int max = ...` (local) with `#define MAX 100` (macro). The case difference makes them legal but the visual similarity is a foot-gun. Linters catch this; the exercise builds one from scratch.

## Try it

1. Extend the program to also flag *near-misses* - identifiers within Levenshtein distance 1 of each other (e.g. `count` vs `counts`).
2. Print the count of clashes per file, sorted.
3. Modify so it only reports if BOTH the lowercased AND uppercased forms appear in the input - that's the more interesting bug pattern.

## Notes from the author

- The "lowercase-as-key" pattern is a common normalisation step in indexing/searching. Most search engines lowercase before tokenising; same idea.
- Storing *all* original spellings, rather than just "did we see a different case?", makes the diagnostic actionable - the user knows which spellings to unify.
- This kind of lint is what `clang-tidy`'s `readability-identifier-naming` catches at scale. Building it teaches the basic indexing skeleton that runs in every linter.
