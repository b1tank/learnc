> **Exercise 1-19.** Write a function `reverse(s)` that reverses the character string `s`. Use it in a program that reverses its input one line at a time.

## Approach

In-place two-pointer swap: index `i` walks right from 0, index `j` walks left from `len - 1`. Swap and step until they meet. Be careful: the line as we receive it includes a trailing `\n`; reversing that would put the newline at the *start* of the output, which isn't what anyone wants. Strip it first, reverse, then re-emit it.

#include <string.h>

## Solution

```c
#include <stdio.h>
#include <string.h>

#define MAXLINE 1000

int read_line(char s[], int maxsize) {
    int c, i = 0;
    while (i < maxsize - 1 && (c = getchar()) != EOF && c != '\n')
        s[i++] = c;
    if (c == '\n') s[i++] = '\n';
    s[i] = '\0';
    return i;
}

void reverse(char s[]) {
    int i = 0;
    int j = (int)strlen(s) - 1;
    while (i < j) {
        char tmp = s[i];
        s[i] = s[j];
        s[j] = tmp;
        ++i;
        --j;
    }
}

int main(void) {
    char line[MAXLINE];
    int  len;

    while ((len = read_line(line, MAXLINE)) > 0) {
        /* peel the trailing newline, if any */
        int had_nl = (len > 0 && line[len - 1] == '\n');
        if (had_nl) line[--len] = '\0';

        reverse(line);

        fputs(line, stdout);
        if (had_nl) putchar('\n');
    }
    return 0;
}
```

## Notes from the author

- `strlen` is in `<string.h>` and walks the string until the `'\0'`. It's O(n) every time you call it. For a one-shot reverse that's fine; in tight loops, cache the length once.
- The two-pointer swap is the canonical pattern; you'll see it again for palindrome checks, array rotations, and a hundred other places. Internalise it early.
- An equivalent recursive version is fun: swap ends, recurse on the inner substring. It's beautiful but blows up the stack on long lines. Iterative wins.
- For Unicode-aware reversal (think emoji, combining characters), this *will* corrupt your output - bytes inside a multibyte character get separated. That's a chapter-7+ concern. For ASCII it's perfect.
