> Write `strend(s, t)` that returns 1 if the string `t` occurs at the **end** of `s`, and 0 otherwise.

## Approach

Compute lengths. If `t` is longer than `s`, it can't be a suffix → return 0. Otherwise, advance `s` so it points to the position `len(s) - len(t)`, then compare from there to the end.

```c:starter
#include <stdio.h>
#include <string.h>

int strend(const char *s, const char *t);

int main(void) {
    printf("%d\n", strend("hello world", "world"));   /* 1 */
    printf("%d\n", strend("hello world", "Hello"));   /* 0 */
    printf("%d\n", strend("foobar",      "bar"));     /* 1 */
    printf("%d\n", strend("foo",         "foobar"));  /* 0 */
    printf("%d\n", strend("anything",    ""));        /* 1 */
    return 0;
}

int strend(const char *s, const char *t) {
    size_t ls = strlen(s);
    size_t lt = strlen(t);
    if (lt > ls)
        return 0;
    s += ls - lt;                  /* jump to the candidate suffix start */
    while (*s && *t) {
        if (*s++ != *t++)
            return 0;
    }
    return 1;
}
```

```output
1
0
1
0
1
```

## Notes from the author

- The empty-`t` case returns 1 — every string ends with the empty string. That's the mathematically clean convention; some implementations choose to return 0. Document whichever you pick.
- Pointer arithmetic on `s += ls - lt;` is exactly the "skip past the prefix" operation. After the jump, `s` and `t` should be equal as strings of length `lt`.
- The Python equivalent is `s.endswith(t)`. Many languages provide it; in C you write it yourself. The exercise is in the pointer arithmetic — note how cleanly the offset computation falls out.
