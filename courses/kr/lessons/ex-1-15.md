> **Exercise 1-15.** Refactor the temperature conversion program of §1.2 so the F-to-C formula lives in its own function.

## Approach

Pull the expression `(5.0/9.0) * (fahr - 32)` out of `printf` and into a `double fahr_to_celsius(int fahr)` function. The body of `main` becomes a clean iteration plus a call.

The point of this exercise isn't algorithmic — it's the smallest possible illustration of the §1.7 message: *functions are how you name a calculation and reuse it.*

```c:starter
#include <stdio.h>

int main(void) {
    int fahr;
    /* TODO: call a function for the conversion */
    for (fahr = 0; fahr <= 300; fahr += 20)
        printf("%3d %6.1f\n", fahr, /* ??? */ 0.0);
    return 0;
}
```

## Solution

```c
#include <stdio.h>

double fahr_to_celsius(int fahr);   /* prototype */

int main(void) {
    printf("%3s %6s\n", "Fahr", "Celsius");
    for (int fahr = 0; fahr <= 300; fahr += 20)
        printf("%3d %6.1f\n", fahr, fahr_to_celsius(fahr));
    return 0;
}

double fahr_to_celsius(int fahr) {
    return (5.0 / 9.0) * (fahr - 32);
}
```

## Notes from the author

- The function is one expression but the *interface* is the gain. Now `fahr_to_celsius(72)` can be called from anywhere, tested in isolation, and renamed in one place when business logic changes.
- The prototype above `main` is good practice even though the function fits on one line. It lets you put `main` first (the public face of the program) and the implementation details below.
- A small style point: `(5.0 / 9.0)` rather than `5.0/9.0`. Spaces around binary operators make formulas easier to scan. The compiler doesn't care.
- A natural sibling is `double celsius_to_fahrenheit(int cel)`. Add it and refactor ex-1-4 to use both.
