> Write `escape(s, t)` that copies the string `t` to `s`, converting non-printable characters like newline and tab into visible escape sequences (`\n`, `\t`, …). Write `unescape(s, t)` that does the reverse.

## Approach

Both functions are character-by-character translators. `escape` reads one char from `t`, writes one or two chars into `s`; `unescape` reads `\` plus its follower from `t`, writes one char into `s`.

Use a `switch` for the translation table - exactly the case it's designed for.

```c:starter
#include <stdio.h>

void escape(char s[], const char t[]);
void unescape(char s[], const char t[]);

int main(void) {
    char raw[]    = "hi\tthere\nfriend";
    char visible[64];
    char restored[64];

    escape(visible, raw);
    unescape(restored, visible);

    printf("raw      : %s\n", raw);
    printf("escaped  : %s\n", visible);
    printf("restored : %s\n", restored);
    return 0;
}
```

## Solution

```c
void escape(char s[], const char t[]) {
    int i = 0, j = 0;
    while (t[i] != '\0') {
        switch (t[i]) {
            case '\n': s[j++] = '\\'; s[j++] = 'n';  break;
            case '\t': s[j++] = '\\'; s[j++] = 't';  break;
            case '\\': s[j++] = '\\'; s[j++] = '\\'; break;
            case '\"': s[j++] = '\\'; s[j++] = '"';  break;
            default:   s[j++] = t[i]; break;
        }
        ++i;
    }
    s[j] = '\0';
}

void unescape(char s[], const char t[]) {
    int i = 0, j = 0;
    while (t[i] != '\0') {
        if (t[i] == '\\') {
            switch (t[++i]) {
                case 'n':  s[j++] = '\n'; break;
                case 't':  s[j++] = '\t'; break;
                case '\\': s[j++] = '\\'; break;
                case '"':  s[j++] = '"';  break;
                case '\0': s[j++] = '\\'; --i; break;   /* trailing '\' */
                default:   s[j++] = '\\'; s[j++] = t[i]; break;
            }
        } else {
            s[j++] = t[i];
        }
        ++i;
    }
    s[j] = '\0';
}
```

```output
raw      : hi	there
friend
escaped  : hi\tthere\nfriend
restored : hi	there
friend
```

## Notes from the author

- The `switch` here is exactly the right tool: a small fixed set of character-to-output mappings, with a default for "no translation". `if`/`else if` would be more verbose and slower.
- Watch the buffer-size contract. `escape` can write up to **twice** as many characters as it reads (every input becomes two output bytes in the worst case). In production, take a size parameter.
- The `case '\0'` branch in `unescape` handles the corner case where `t` ends with a bare `\`. Without it, you'd index past the null terminator into garbage. Always think about the malformed-input case in parsers, even tiny ones.
