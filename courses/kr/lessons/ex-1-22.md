> **Exercise 1-22.** Write a program that folds long input lines into two or more shorter lines after the last non-blank character that occurs before the *n*th column of input. Handle very long lines and lines with no blanks until past column *n*.

## Approach

Read characters into a buffer. When the buffer length hits the fold column, look *backward* for the last blank. If you find one, break there: emit everything up to it, then continue with what follows. If there is no blank, break at the column anyway (you can't do better without distorting the data).

```c:starter
#include <stdio.h>

#define FOLD 30
#define MAXLINE 1000

int main(void) {
    /* TODO: fold long lines */
    return 0;
}
```

## Solution

```c
#include <stdio.h>

#define FOLD     30
#define MAXLINE  1000

static char buf[MAXLINE];
static int  len = 0;

static void flush_through(int upto) {
    /* emit buf[0..upto), then shift the rest to the front */
    for (int i = 0; i < upto; ++i) putchar(buf[i]);
    putchar('\n');
    int rest = len - upto;
    /* skip a leading space at the fold point so the next line doesn't start with blank */
    int skip = 0;
    while (skip < rest && buf[upto + skip] == ' ') ++skip;
    for (int i = 0; i < rest - skip; ++i) buf[i] = buf[upto + skip + i];
    len = rest - skip;
}

int main(void) {
    int c;

    while ((c = getchar()) != EOF) {
        if (c == '\n') {
            for (int i = 0; i < len; ++i) putchar(buf[i]);
            putchar('\n');
            len = 0;
            continue;
        }

        if (len < MAXLINE - 1) buf[len++] = (char)c;

        if (len >= FOLD) {
            /* find the last blank in buf[0..len) */
            int b = len - 1;
            while (b >= 0 && buf[b] != ' ') --b;
            if (b > 0) flush_through(b);
            else       flush_through(len);   /* no blank - hard break */
        }
    }

    /* drain remainder */
    for (int i = 0; i < len; ++i) putchar(buf[i]);
    if (len > 0) putchar('\n');
    return 0;
}
```

```output
```

Input (one line, `FOLD = 30`):

```
the quick brown fox jumps over the lazy dog
```

Output:

```
the quick brown fox jumps
over the lazy dog
```

The first fold happened after `jumps` (the last blank before column 30).

## Notes from the author

- The data structure here is one buffer that grows to at most `FOLD` characters before each decision. That's why it's important to act *at* FOLD, not after - otherwise the buffer can grow unboundedly on a single blankless mega-token.
- The "skip a leading blank after the fold" step (`while (skip < rest && buf[upto + skip] == ' ')`) is a quality nicety: avoids a fold like `"long line \n word"` becoming `"long line\n word"` with a leading space on line two.
- This pattern - emit-then-shift - is O(n²) in the worst case because of all the shifts. A real implementation uses a *ring buffer* or a sliding window. For input lines under a few KB it's irrelevant.
- The standard tool `fmt -w 30` does essentially this. POSIX `fold -s -w 30` is closer still. Compare your output against both.
