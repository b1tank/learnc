> **Exercise 2-2.** The line-reader in §1.9 has the loop condition
> `i < lim - 1 && (c = getchar()) != '\n' && c != EOF`.
> Rewrite the loop without using `&&` or `||`.

## Approach

`&&` and `||` are doing two jobs: combining tests *and* short-circuiting. Without them you need:

1. **Nested `if`s** to express the "and" relationship, and
2. **`break`** to exit when any one condition fails - which restores the short-circuit guarantee that the other tests don't run.

```c:starter
#include <stdio.h>

#define MAXLINE 100

int read_line(char s[], int lim);

int main(void) {
    char line[MAXLINE];
    int  len = read_line(line, MAXLINE);
    printf("read %d chars: %s", len, line);
    return 0;
}

int read_line(char s[], int lim) {
    int i = 0;
    /* TODO: rewrite the original three-clause && loop */
    return i;
}
```

## Solution

```c
int read_line(char s[], int lim) {
    int c, i;

    for (i = 0; i < lim - 1; ++i) {
        c = getchar();
        if (c == EOF)  break;
        if (c == '\n') break;
        s[i] = c;
    }
    /* Save the newline if we got one, before the buffer cap */
    if (i < lim - 1 && c == '\n') {
        s[i++] = '\n';
    }
    s[i] = '\0';
    return i;
}
```

Each `break` is a short-circuit: once we've decided to exit, the next test doesn't run. The bare `if (cond) break;` chain is exactly what `&&` did, just spelled out.

## Notes from the author

- This exercise looks contrived but it points at something real: `&&`/`||` are *desugared* by the compiler into branches with short-circuit jumps. The hand-rewritten version is what your `if`-chain becomes in the generated assembly anyway. Seeing it spelled out helps when you're reading optimized output.
- Notice the original `&&` version reads the buffer-bound check first, then the input, then the newline test. The expanded version preserves that order - getchar() does not run if the buffer is full. Swapping the order would change behavior (reading one too many characters).
- A common follow-up: rewrite with a `while (1)` and one `break` for each exit condition. Same shape, different keyword.
