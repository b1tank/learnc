> Write `expand(s1, s2)` that expands shorthand notation like `a-z` in `s1` into the equivalent complete string `abcdef...xyz` in `s2`. Allow letters and digits, both cases, and handle a leading or trailing `-` literally (since there's no other character on the relevant side).

## Approach

Walk `s1`. For each character, peek at the next two: if the pattern is `X-Y` where `X < Y` and the two are in the same "class" (both lowercase, both uppercase, or both digits), emit every character from `X` up through `Y`. Otherwise emit the character literally.

A leading `-` (e.g. `-a`) or trailing `-` (e.g. `a-`) is treated as a literal hyphen - no range starts/ends with nothing.

```c:starter
#include <stdio.h>
#include <ctype.h>

void expand(const char s1[], char s2[]);

int main(void) {
    char out[256];

    expand("a-e", out);          printf("[a-e] -> %s\n", out);
    expand("a-c0-3X-Z", out);    printf("[a-c0-3X-Z] -> %s\n", out);
    expand("-a-z-", out);        printf("[-a-z-] -> %s\n", out);
    expand("z-a", out);          printf("[z-a] -> %s\n", out);  /* not a valid range */
    return 0;
}
```

## Solution

```c
static int same_class(int a, int b) {
    if (islower(a) && islower(b)) return 1;
    if (isupper(a) && isupper(b)) return 1;
    if (isdigit(a) && isdigit(b)) return 1;
    return 0;
}

void expand(const char s1[], char s2[]) {
    int i = 0, j = 0;
    while (s1[i] != '\0') {
        char a = s1[i];
        if (a == '-' && i > 0 && s1[i+1] != '\0') {
            /* '-' between two chars: handled by the previous iteration's lookahead */
            s2[j++] = a;            /* shouldn't be reached if lookahead worked */
            ++i;
        } else if (s1[i+1] == '-' && s1[i+2] != '\0' && same_class(a, s1[i+2]) && a < s1[i+2]) {
            for (char c = a; c <= s1[i+2]; ++c)
                s2[j++] = c;
            i += 3;                 /* consumed X, -, Y */
        } else {
            s2[j++] = a;
            ++i;
        }
    }
    s2[j] = '\0';
}
```

```output
[a-e] -> abcde
[a-c0-3X-Z] -> abc0123XYZ
[-a-z-] -> -abcdefghijklmnopqrstuvwxyz-
[z-a] -> z-a
```

## Notes from the author

- Three-character lookahead (`s1[i]`, `s1[i+1]`, `s1[i+2]`) is fine when the string is null-terminated - the test `s1[i+1] != '\0' && s1[i+2] != '\0'` guarantees you don't read off the end. But chain the `&&` checks left-to-right (C short-circuits).
- The `same_class` helper is what keeps `A-z` from generating the awful run `ABC...XYZ[\]^_`abc...xyz` of all ASCII characters between `Z` and `a`. ASCII has six punctuation chars between the two letter blocks, and you almost never want those.
- The `a < s1[i+2]` test rejects reversed ranges like `z-a` (treats them literally instead of producing nothing). The classical alternative is to swap and reverse - that breaks the "expand in order" contract, so I prefer the literal-fallback approach.
