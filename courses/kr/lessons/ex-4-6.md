> Give the calculator 26 single-letter variables `A`-`Z` so you can store and recall values. Also add a register that holds the most recent printed value (so the user can re-use it).

## Approach

- A 26-element array `double vars[26]` indexed by `name - 'A'`.
- A single `double last` holding the most recent printed result.
- New commands:
  - `<letter> =` — pop, store into the variable
  - `<letter>` (alone) — push the stored value
  - `_` (underscore) — push `last`

```c:starter
#include <stdio.h>
#include <ctype.h>
#include <stdlib.h>

#define MAXVAL 100
int    sp = 0;
double val[MAXVAL];
double vars[26] = { 0 };
double last = 0.0;

void   push(double f) { if (sp < MAXVAL) val[sp++] = f; }
double pop(void)      { return (sp > 0) ? val[--sp] : 0.0; }

void store_var(char letter) {
    if (!isupper((unsigned char)letter)) return;
    if (sp > 0) vars[letter - 'A'] = val[sp-1];  /* peek, don't pop */
}
void load_var(char letter) {
    if (!isupper((unsigned char)letter)) return;
    push(vars[letter - 'A']);
}
void print_top(void) {
    if (sp > 0) {
        last = val[sp-1];
        printf("%g\n", last);
    }
}

int main(void) {
    push(42);     store_var('A');     /* A = 42 */
    push(3.14);   store_var('B');     /* B = 3.14 */

    load_var('A');                    /* push 42 */
    load_var('B');                    /* push 3.14 */
    push(pop() + pop());              /* 45.14 */
    print_top();                      /* prints 45.14, also stores last */

    push(last);
    push(2);
    push(pop() * pop());              /* last * 2 */
    print_top();                      /* 90.28 */
    return 0;
}
```

```output
45.14
90.28
```

## Notes from the author

- 26 variables is a fixed table small enough to fit in registers and L1 cache; the lookup `letter - 'A'` is a single subtraction. Going to "arbitrary string variable names" means hashing, allocation, and a small-string-optimised table — much more code.
- The "store-on-peek" choice (don't pop, just store the current top) means `A= A= A=` doesn't drain the stack, which feels natural for an interactive user. Some calculators choose pop-and-store; it's a UX decision.
- The `last` register is a small detail that makes a calculator feel alive. Every desktop calculator has it (often labelled `Ans`). Adding it costs five lines but is the kind of polish that distinguishes a tutorial example from a usable tool.
