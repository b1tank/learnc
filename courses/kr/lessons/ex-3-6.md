> Extend `itoa` to take a third argument: the minimum field width. The result should be right-justified, padded on the left with spaces so the total length is at least the given width.

## Approach

Three steps:

1. Convert `n` into its base-10 form in a temporary buffer (same as plain `itoa`).
2. Measure its length `L`.
3. If `L < width`, write `width - L` spaces into `s`, then copy the digits. Otherwise just copy the digits.

```c:starter
#include <stdio.h>
#include <string.h>
#include <limits.h>

void itoa_w(int n, char s[], int width);

int main(void) {
    char buf[64];

    itoa_w(42,       buf, 6);  printf("[%s]\n", buf);   /* "    42" */
    itoa_w(-17,      buf, 6);  printf("[%s]\n", buf);   /* "   -17" */
    itoa_w(12345,    buf, 3);  printf("[%s]\n", buf);   /* "12345" (no clipping) */
    itoa_w(INT_MIN,  buf, 4);  printf("[%s]\n", buf);   /* "-2147483648" */
    itoa_w(0,        buf, 5);  printf("[%s]\n", buf);   /* "    0" */
    return 0;
}
```

## Solution

```c
static void reverse(char s[], int from, int to) {
    while (from < to) {
        char t = s[from]; s[from] = s[to]; s[to] = t;
        ++from; --to;
    }
}

void itoa_w(int n, char s[], int width) {
    char tmp[32];
    int sign = n;
    int i = 0;

    /* convert into tmp, least-significant digit first */
    do {
        int d = n % 10;
        if (d < 0) d = -d;
        tmp[i++] = d + '0';
        n /= 10;
    } while (n != 0);

    if (sign < 0)
        tmp[i++] = '-';
    /* tmp now holds the digits in reverse order; length is i */

    int pad = width - i;
    int k = 0;
    while (pad-- > 0)
        s[k++] = ' ';

    /* copy tmp into s in correct (forward) order */
    while (i > 0)
        s[k++] = tmp[--i];

    s[k] = '\0';
}
```

```output
[    42]
[   -17]
[12345]
[-2147483648]
[    0]
```

## Notes from the author

- Building the digits in a temporary buffer first (then padding + copying) is conceptually cleaner than the alternative (write to `s` directly, then shift everything right by `pad`). The shift approach is `O(n)` per pad-character; the buffer approach is `O(n)` total.
- When the value is already wider than `width`, we **don't truncate** — we just print it. That matches the behaviour of `printf("%6d", n)`. Truncating a number silently is almost always a bug.
- This entire exercise is one `printf("%*d", width, n)` call in production. Re-implementing it teaches you how `printf`'s width spec works underneath; that knowledge pays off the first time you have to write something `printf` can't directly do (locale-aware grouping, custom padding char, etc.).
