> **Exercise 1-9.** Copy stdin to stdout, but whenever you encounter two or more consecutive blanks (`' '`), emit just one. Every other character passes through unchanged.

## Approach

A one-character lookback is enough. Track whether the *previous* character was a blank. If the current one is a blank and the previous one was too, drop it; otherwise emit it.

```c:starter
#include <stdio.h>

int main(void) {
    /* TODO: copy stdin to stdout, collapsing blank runs */
    return 0;
}
```

## Solution

```c
#include <stdio.h>

int main(void) {
    int c, prev = 0;   /* 0 is never a legal byte from a text stream */

    while ((c = getchar()) != EOF) {
        if (c == ' ' && prev == ' ')
            ;                /* skip this redundant blank */
        else
            putchar(c);
        prev = c;
    }
    return 0;
}
```

```output
```

Try input `the    quick   brown  fox`. Output: `the quick brown fox`.

## Notes from the author

- The `prev = 0;` initialisation is a small trick: zero isn't a legal character from a text stream (well, `'\0'` is but doesn't appear in normal input), so the first character can never spuriously match `prev`. You could also use `prev = EOF`; same effect.
- The empty-then-assign style (`if (skip) ; else emit;`) reads slightly awkwardly. A cleaner form is `if (!(c == ' ' && prev == ' ')) putchar(c);` - flip the predicate. Both compile to the same code.
- Generalising: a *state machine* with two states ("just-saw-blank" vs "not"). For richer rules (collapse any whitespace run, normalise blanks-and-tabs to one space), bump up to an explicit `enum`.
- The standard tool that does this is `tr -s ' '` in Unix. Try comparing your output against `tr` on a long file.
