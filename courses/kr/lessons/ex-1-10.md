> **Exercise 1-10.** Copy stdin to stdout, but replace tabs with the literal two characters `\t`, backspaces with `\b`, and backslashes with `\\`. The intent is to produce text where every special character is *visible* — useful when you want to inspect data that contains them.

## Approach

Per-character switch. Each problem character maps to a two-character output sequence.

```c:starter
#include <stdio.h>

int main(void) {
    /* TODO: make special chars visible */
    return 0;
}
```

## Solution

```c
#include <stdio.h>

int main(void) {
    int c;

    while ((c = getchar()) != EOF) {
        switch (c) {
            case '\t': putchar('\\'); putchar('t'); break;
            case '\b': putchar('\\'); putchar('b'); break;
            case '\\': putchar('\\'); putchar('\\'); break;
            default:   putchar(c);
        }
    }
    return 0;
}
```


Input: a literal tab between `foo` and `bar`. Output: `foo\tbar`.

## Notes from the author

- This is the `cat -A` / `od -c` / `echo -e` family of tools in miniature — they exist precisely because invisible bytes inside text files are a constant debugging headache.
- I used `switch` to enumerate the special cases; an equivalent chain of `if`/`else if` works too. With three cases either is fine; with twenty, `switch` wins on clarity.
- Watch out: the *input* `\t` is a single byte (`0x09`). The *output* `\t` is the two literal characters backslash and lowercase-t. The program is bridging between two representations of the same logical idea.
- Stretch: extend to handle all C escapes (`\n`, `\r`, `\v`, `\f`, `\a`, `\0`) and fall back to `\xNN` (hex) for anything else non-printable. That's a great §2 follow-up once `printf("\\x%02x", c)` makes sense.
