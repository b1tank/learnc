> **Exercise 1-14.** Print a histogram of how often each character appears in the input.

## Approach

A 128-element array indexed by character code is the right data structure for ASCII. Index `c` directly - `++count[c]`. Iterate the array afterwards, skipping zeros and unprintable bytes.

```c:starter
#include <stdio.h>

int main(void) {
    /* TODO: character-frequency histogram */
    return 0;
}
```

## Solution

```c
#include <stdio.h>

#define NCHAR 128

int main(void) {
    int c;
    int count[NCHAR] = {0};

    while ((c = getchar()) != EOF) {
        if (c >= 0 && c < NCHAR)
            ++count[c];
    }

    for (int i = 0; i < NCHAR; ++i) {
        if (count[i] == 0) continue;
        if (i == ' ')      printf("' ' (sp)  | ");
        else if (i == '\t') printf("'\\t'     | ");
        else if (i == '\n') printf("'\\n'     | ");
        else if (i < 32 || i == 127) printf("0x%02x     | ", i);
        else                printf("'%c'      | ", (char)i);
        for (int j = 0; j < count[i]; ++j) putchar('#');
        printf(" %d\n", count[i]);
    }

    return 0;
}
```

## Notes from the author

- The `if (c >= 0 && c < NCHAR)` guard matters because `getchar()` returns `int` and the standard does not say byte values are positive. On platforms where `char` is signed, a byte ≥ 128 comes back as a negative `int`, which would index out of bounds. Modern wisdom is to cast: `int idx = (unsigned char) c; ++count[idx];`.
- For full-byte coverage replace `128` with `256` and drop the unicode discussion until chapter 7.
- The bars get long fast. A scaling pass (`max_count` → 60 columns) is the next natural extension.
- This same shape is the start of an *entropy* calculation. Once you have the counts, `-Σ p_i log p_i` over the non-zero buckets gives Shannon entropy of the input - handy for spotting random data vs structured text. Try it once the math feels ready.
