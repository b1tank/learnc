> **Exercise 1-5.** Modify the temperature conversion program to print the table backwards — from 300°F down to 0°F.

## Approach

Three things in the `for` header have to flip:

1. **Init** — start at the upper bound instead of the lower bound.
2. **Test** — stop when the counter goes *below* the lower bound, not *above* the upper one.
3. **Update** — decrement instead of increment.

That's it. The body of the loop never changes.

```c:starter
#include <stdio.h>

int main(void) {
    int fahr;

    /* TODO: reverse direction */
    for (fahr = 0; fahr <= 300; fahr += 20)
        printf("%3d %6.1f\n", fahr, (5.0/9.0) * (fahr - 32));

    return 0;
}
```

## Solution

```c
#include <stdio.h>

int main(void) {
    int fahr;

    printf("%3s %6s\n", "Fahr", "Celsius");
    for (fahr = 300; fahr >= 0; fahr -= 20)
        printf("%3d %6.1f\n", fahr, (5.0/9.0) * (fahr - 32));

    return 0;
}
```

## Notes from the author

- Notice that none of the body changed — the table is the same data, just listed in reverse. A clean illustration of how the `for` header completely abstracts iteration direction.
- A subtle bug to watch for: if you'd kept `fahr += 20` while flipping init and test, the loop would run zero times because `300 + 20 > 300` immediately. The update direction has to flip *with* the init/test pair, not independently.
- Stretch: take the loop direction as a command-line argument. We'll see `argv` in §5.10; for now, parameterise by a `#define DIR -1`-style switch and use it in both the init and update slots.
