> Write a program that prints a set of files, each starting on a new page, with the filename and page number as a heading.

## Approach

Pick a page height (66 lines is a classic for 11" paper; we'll use 60 for the body, plus the heading). For each file:

1. Reset the page counter to 1.
2. Print the heading (filename + page 1).
3. Read lines; after every `LINES_PER_PAGE` lines, emit a form-feed (or just a newline group) and a new heading with incremented page number.

```c:starter
#include <stdio.h>

#define LINES_PER_PAGE 60

static void print_heading(const char *name, int page) {
    printf("\n========================================\n");
    printf("File: %s     Page %d\n", name, page);
    printf("========================================\n\n");
}

static void print_file(const char *name) {
    FILE *fp = fopen(name, "r");
    if (!fp) { perror(name); return; }

    int page  = 1;
    int line  = 0;
    print_heading(name, page);

    char buf[4096];
    while (fgets(buf, sizeof buf, fp)) {
        fputs(buf, stdout);
        ++line;
        if (line >= LINES_PER_PAGE) {
            putchar('\f');           /* form feed */
            ++page;
            line = 0;
            print_heading(name, page);
        }
    }
    /* pad final page with newlines to keep file separation visible */
    while (line < LINES_PER_PAGE) { putchar('\n'); ++line; }
    fclose(fp);
}

int main(int argc, char *argv[]) {
    if (argc < 2) {
        fprintf(stderr, "usage: %s FILE [FILE...]\n", argv[0]);
        return 2;
    }
    for (int i = 1; i < argc; ++i)
        print_file(argv[i]);
    return 0;
}
```

## Form feed (`\f`)

`\f` is ASCII 12. On a printer (back when these were common), it advanced the paper to the top of the next page. On a terminal, it's mostly a no-op. The print tradition is preserved - `lp` and `enscript` still emit form feeds.

## Try it

1. Add a `-l N` option to override the lines-per-page.
2. Wrap long lines at column 80 instead of letting them overflow.
3. Add a footer with date and time using `time()` / `strftime()`.

## Notes from the author

- "Page" used to mean "physical printed page". `pr(1)` and `enscript(1)` are the legacy Unix tools for this. Today most people render to PDF instead, but the line-counting logic is identical.
- The `\f` is invisible on terminals but real on printers. Send the output to a USB-connected printer with `lp` and you'll see the pagination in action.
- Adding line numbers (`%4d %s`) is a small extension that produces beautiful source-code listings. `enscript -B -2 --highlight=c file.c` does this with syntax highlighting.
