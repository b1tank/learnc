> The §4.3 RPN calculator did its own tokenising. Rewrite it using `scanf` (or `sscanf` on lines read with `fgets`) so the standard library does the parsing.

## Approach

Read a token at a time. `scanf("%lf", &v)` parses a number; if it returns 0, the next non-whitespace character is an operator — fetch it with `scanf(" %c", &op)`.

This works because `scanf("%lf", ...)` returns 0 if the next non-whitespace character can't start a number; the buffer is left intact so the next call can read the operator.

```c:starter
#include <stdio.h>
#include <stdlib.h>

#define MAXVAL 100
static double val[MAXVAL];
static int    sp = 0;

static void   push(double v) { if (sp < MAXVAL) val[sp++] = v; }
static double pop(void)      { return sp > 0 ? val[--sp] : 0; }

int main(void) {
    double v;
    char op;
    for (;;) {
        int rc = scanf("%lf", &v);
        if (rc == 1) {
            push(v);
        } else if (rc == 0) {
            if (scanf(" %c", &op) != 1) break;
            switch (op) {
                case '+': push(pop() + pop()); break;
                case '*': push(pop() * pop()); break;
                case '-': { double b = pop(), a = pop(); push(a - b); break; }
                case '/': { double b = pop(), a = pop(); push(b ? a/b : 0); break; }
                case '\n':
                case 'q': case '=':
                    printf("= %g\n", sp ? val[sp - 1] : 0.0);
                    if (op == 'q') return 0;
                    break;
                default:
                    fprintf(stderr, "unknown op '%c'\n", op);
            }
        } else {
            break;  /* EOF */
        }
    }
    return 0;
}
```

## Why this is cleaner

The textbook version had ~30 lines of state-machine code for tokenising. Here, `scanf` does it. The trade-off: less control over error recovery — if `scanf` chokes on an unexpected character, you have to flush it manually.

## Try it

1. Replace the loop with `fgets` + `sscanf` so you can re-read on error.
2. Add a "stack" command that prints the current stack contents.
3. Make `=` not consume; print without popping.

## Notes from the author

- The "rc == 0 means try next conversion" idiom is one of `scanf`'s few elegant features. Combined with `" %c"` (whitespace-skipping single char), you get a tiny token reader.
- For a production calculator, you'd want `fgets` line-at-a-time and proper error recovery. `scanf` directly on `stdin` is fragile.
- The `unsigned char` cast and `ungetc` dance from ex-5-1 isn't needed here because `scanf` handles whitespace and pushback internally.
