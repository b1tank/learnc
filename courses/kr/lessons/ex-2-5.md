> **Exercise 2-5.** Write `any(s1, s2)`, which returns the first position in `s1` where any character from `s2` occurs, or `-1` if no character of `s2` appears in `s1`.

## Approach

Same nested-scan as squeeze, except now we *return on the first hit* instead of compacting.

```c:starter
#include <stdio.h>

int any(const char s1[], const char s2[]);

int main(void) {
    printf("%d\n", any("hello, world", "xyz"));    /* -1 */
    printf("%d\n", any("hello, world", "wd"));     /* 7 */
    printf("%d\n", any("hello, world", ","));      /* 5 */
    return 0;
}

int any(const char s1[], const char s2[]) {
    /* TODO */
    return -1;
}
```

## Solution

```c
int any(const char s1[], const char s2[]) {
    for (int i = 0; s1[i] != '\0'; ++i) {
        for (int j = 0; s2[j] != '\0'; ++j) {
            if (s1[i] == s2[j]) return i;
        }
    }
    return -1;
}
```

```output
-1
7
5
```

## Notes from the author

- This is the same algorithm `strpbrk` from `<string.h>` implements (returning a pointer instead of an index, and `NULL` for "no match"). For real code, prefer the standard function — it's tested, fast, and often vectorised.
- The double loop is $O(|s1| \cdot |s2|)$. For tiny `s2` that's fine. For "scan a large file for any of 500 bad keywords," you want a more clever data structure — a 256-byte lookup table for single-character matches, or Aho–Corasick for multi-character patterns.
- Returning `-1` for "not found" is a K&R-era convention that's now considered awkward because the return value mixes "an index" and "an error code". Modern style: return `size_t` and use a separate `bool found` out-parameter, or return a pointer (`NULL` for not found) like `strpbrk` does. Either way, the caller has to check.
