> Give the RPN calculator access to library math functions like `sin`, `cos`, `exp`, `pow`. The first three take one argument (pop one, push the result); `pow` takes two (pop two, push the result).

## Approach

Add a token-dispatch branch that compares the incoming token string against the function names. Pop the required number of arguments, call the C library function, push the result.

```c:starter
#include <stdio.h>
#include <math.h>
#include <string.h>

#define MAXVAL 100
int    sp = 0;
double val[MAXVAL];

void   push(double f) { if (sp < MAXVAL) val[sp++] = f; }
double pop(void)      { return (sp > 0) ? val[--sp] : 0.0; }

/* dispatch a math function by name */
int apply_math(const char *name) {
    if (strcmp(name, "sin") == 0)   { push(sin(pop()));   return 1; }
    if (strcmp(name, "cos") == 0)   { push(cos(pop()));   return 1; }
    if (strcmp(name, "exp") == 0)   { push(exp(pop()));   return 1; }
    if (strcmp(name, "log") == 0)   { push(log(pop()));   return 1; }
    if (strcmp(name, "sqrt") == 0)  { push(sqrt(pop()));  return 1; }
    if (strcmp(name, "pow") == 0) {
        double y = pop();
        double x = pop();
        push(pow(x, y));
        return 1;
    }
    return 0;
}

int main(void) {
    push(0.0);              apply_math("sin");  printf("sin(0)     = %g\n", pop());
    push(0.0);              apply_math("cos");  printf("cos(0)     = %g\n", pop());
    push(1.0);              apply_math("exp");  printf("exp(1)     = %g\n", pop());
    push(2.0); push(10.0);  apply_math("pow");  printf("pow(2,10)  = %g\n", pop());
    push(2.0);              apply_math("sqrt"); printf("sqrt(2)    = %g\n", pop());
    return 0;
}
```

```output
sin(0)     = 0
cos(0)     = 1
exp(1)     = 2.71828
pow(2,10)  = 1024
sqrt(2)    = 1.41421
```

## Don't forget `-lm`

On Linux you must explicitly link the math library:

```bash
gcc calc.c -o calc -lm
```

This is a small surprise - the standard library's I/O functions are auto-linked, but the math functions sit in a separate `libm.a`. The "linker symbol not found: sin" error means "you forgot -lm". macOS and modern Windows compilers auto-link it; Linux's `gcc` keeps the legacy convention.

## Notes from the author

- The "two-argument `pop` order" trap from ex-4-3 applies again. For `pow(x, y)`, pop `y` first (it was pushed last), then `x`. Get the order wrong and you'll compute `pow(y, x)` silently.
- Each math function has edge cases: `sqrt(-1)` returns NaN, `log(0)` returns -infinity, `pow(0, 0)` is 1 by IEEE convention. Decide whether you want to surface those or filter them.
- A small string-dispatch table (`{"sin", sin1arg}, {"pow", pow2arg}, ...`) is a step toward generalising this. For a real calculator, you'd implement a *named-function registry* so adding new functions doesn't touch the dispatch code.
