> Write `tail`, which prints the last `n` lines of its input. The default `n` is 10. The argument `-N` (e.g. `-5`) sets `n` to N.

## Approach

A circular buffer of line pointers, size `n`. As you read lines, overwrite the oldest. After EOF, print starting from the oldest still-alive slot and wrap around.

For storage, each line goes into a heap allocation; when the buffer wraps, free the old line before overwriting.

```c:starter
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

int main(int argc, char *argv[]) {
    int n = 10;
    if (argc >= 2 && argv[1][0] == '-')
        n = atoi(argv[1] + 1);
    if (n <= 0) n = 10;

    /* circular buffer */
    char **lines = calloc(n, sizeof *lines);
    if (!lines) { perror("calloc"); return 1; }

    char buf[4096];
    int  next = 0;          /* write index */
    int  total = 0;          /* number of lines read */

    while (fgets(buf, sizeof buf, stdin)) {
        free(lines[next]);                /* free the oldest in this slot */
        lines[next] = strdup(buf);
        next  = (next + 1) % n;
        ++total;
    }

    /* print from oldest to newest */
    int count = (total < n) ? total : n;
    int start = (total < n) ? 0 : next;   /* if buffer wrapped, oldest is at `next` */
    for (int i = 0; i < count; ++i)
        fputs(lines[(start + i) % n], stdout);

    for (int i = 0; i < n; ++i) free(lines[i]);
    free(lines);
    return 0;
}
```

Usage:

```bash
$ seq 1 100 | ./tail -5
96
97
98
99
100
$ ./tail < /etc/passwd      # default 10
```

## Why a circular buffer

Memory: `O(n)` regardless of input length. Even a 10-gigabyte input keeps the buffer at `n` lines.

Time: `O(1)` per line read (one free, one strdup, one mod). Total `O(total_lines)`.

The alternative — buffering the whole input — works for small files but blows out memory on huge ones. The circular buffer trades a small modular arithmetic cost for unbounded streaming capability.

## Notes from the author

- `strdup` (POSIX, in `<string.h>`) is `malloc + strcpy`. C99 doesn't have it; modern C does. If you can't use it, write `strndup` or your own.
- The wraparound logic — "if the buffer wrapped, the oldest is at `next`, otherwise at index 0" — is the heart of the circular-buffer pattern. Getting it right is the exercise.
- The real `tail(1)` has many more options (`-f` follow, `-c` byte count, `--lines=`). The core mechanism is what we just built; the rest is tactical I/O work.
