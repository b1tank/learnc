> Write a program that converts uppercase to lowercase or vice versa. The direction depends on `argv[0]` - the program's invocation name. For example, link the same binary as both `tolower` and `toupper`; running `./tolower` lowercases, running `./toupper` uppercases.

## Approach

Strip path components from `argv[0]` to get the basename, then compare against `"toupper"` / `"tolower"`.

```c:starter
#include <stdio.h>
#include <ctype.h>
#include <string.h>

int main(int argc, char *argv[]) {
    const char *name = argv[0];
    const char *slash = strrchr(name, '/');
    if (slash) name = slash + 1;        /* skip path */

    int (*convert)(int) = NULL;
    if      (strcmp(name, "toupper") == 0) convert = toupper;
    else if (strcmp(name, "tolower") == 0) convert = tolower;
    else {
        fprintf(stderr, "%s: invoke as 'toupper' or 'tolower'\n", argv[0]);
        return 1;
    }

    int c;
    while ((c = getchar()) != EOF)
        putchar(convert(c));
    return 0;
}
```

```output
(awaits stdin)
```

Compile and link with two names:

```bash
$ gcc -o tolower case_filter.c
$ ln -s tolower toupper
$ echo "Hello, World" | ./tolower
hello, world
$ echo "Hello, World" | ./toupper
HELLO, WORLD
```

## Why argv[0]?

Many Unix tools use the **multi-link** trick: one binary, multiple symlink names, behaviour switches on the invocation name. Classic examples:

- `gunzip` and `zcat` are typically symlinks to `gzip`.
- `vim`, `vi`, `view`, `gvim` all link to the same binary.
- `busybox` provides `ls`, `cat`, `cp`, etc. as symlinks; argv[0] selects which "command" runs.

The trick saves disk space and lets distinct commands share helper code.

## Notes from the author

- `argv[0]` is *conventionally* the program's name, but the shell sets it - there's no kernel guarantee. A weird shell could set it to anything. Don't use `argv[0]` for security decisions.
- `strrchr` (find last occurrence) is the right tool here. `strchr` (first) would give the wrong basename if the path had multiple slashes.
- The function-pointer dispatch (`convert = toupper`) is cleaner than `if`-chains inside the loop. Build the dispatch table once, run the loop fast.
