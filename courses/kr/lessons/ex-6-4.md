> Read text and print each distinct word with its frequency, sorted from most frequent to least.

## Approach

Same hash table as ex-6-3 (word → count). After reading, copy the entries into an array and `qsort` by count descending.

```c:starter
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <ctype.h>

#define HASHSIZE 257

struct word {
    char *w;
    int count;
    struct word *next;
};

static struct word *table[HASHSIZE];
static int total_words = 0;

static unsigned hash(const char *s) {
    unsigned h = 0;
    while (*s) h = 31u * h + (unsigned char)*s++;
    return h % HASHSIZE;
}

static void bump(const char *w) {
    unsigned h = hash(w);
    struct word *e;
    for (e = table[h]; e; e = e->next)
        if (strcmp(e->w, w) == 0) { ++e->count; return; }
    e = malloc(sizeof *e);
    e->w = strdup(w);
    e->count = 1;
    e->next = table[h];
    table[h] = e;
    ++total_words;
}

static int cmp_desc(const void *a, const void *b) {
    const struct word *pa = *(const struct word *const *)a;
    const struct word *pb = *(const struct word *const *)b;
    if (pa->count != pb->count) return pb->count - pa->count;   /* desc count */
    return strcmp(pa->w, pb->w);                                  /* asc word */
}

int main(void) {
    int c;
    char buf[64];
    int bi = 0;
    while ((c = getchar()) != EOF) {
        if (isalpha(c) || c == '_') {
            if (bi < (int)sizeof buf - 1) buf[bi++] = tolower(c);
        } else if (bi > 0) {
            buf[bi] = '\0'; bump(buf); bi = 0;
        }
    }
    if (bi > 0) { buf[bi] = '\0'; bump(buf); }

    struct word **arr = malloc(total_words * sizeof *arr);
    int k = 0;
    for (int i = 0; i < HASHSIZE; ++i)
        for (struct word *e = table[i]; e; e = e->next)
            arr[k++] = e;
    qsort(arr, total_words, sizeof *arr, cmp_desc);

    for (int i = 0; i < total_words; ++i)
        printf("%5d  %s\n", arr[i]->count, arr[i]->w);
    free(arr);
    return 0;
}
```

```output
(feed e.g. "the the the cat the dog cat\n")
```

Output:

```output
    4  the
    2  cat
    1  dog
```

## Why descending by count

Frequency lists answer "what's the most common?" — sorting descending puts the most-common-first. The secondary alphabetic key (when counts tie) gives a stable, deterministic order.

## A nice statistical view

If you feed in any reasonably long English text (say, a chapter of a book), the frequency follows Zipf's law: the n-th most-common word occurs `≈ 1/n` times as often as the most common. A pleasing pattern to verify with this program.

## Try it

1. Limit output to top N words via a CLI argument.
2. Print frequencies as percentages (`23.5%`) instead of raw counts.
3. Detect Zipf's law: print `count[i] * i` for each rank `i` — should be roughly constant.

## Notes from the author

- "Count, sort, print" is the most common shape of a text-mining program. Build it once and reuse the skeleton for log-line frequencies, error counts, etc.
- The `qsort` comparator returns "negative, zero, positive" — be careful with `pb->count - pa->count` if counts can overflow into negative. For 32-bit counts on long input, switch to explicit `<` / `>`.
- The secondary key (alphabetical on ties) ensures the output is the same each run. Without it, tie-broken output depends on hash-table iteration order, which can change with bucket size or insertion order.
