> **Exercise 2-6.** Write `setbits(x, p, n, y)` that returns `x` with the `n` bits starting at position `p` (counting from bit 0 at the right) replaced by the rightmost `n` bits of `y`. All other bits of `x` are unchanged.

## Approach

Three steps:

1. Build a mask of `n` ones, shifted into position `p`.
2. Clear those bits in `x` with `x & ~mask`.
3. Extract the bottom `n` bits of `y`, shift them into position, and OR them into the cleared `x`.

The "n ones" mask is `(1U << n) - 1` - that's a number whose binary form is exactly `n` 1-bits.

```c:starter
#include <stdio.h>

unsigned setbits(unsigned x, int p, int n, unsigned y);

int main(void) {
    /* Replace bits 4..7 of 0xFF00 with the bottom 4 bits of 0x000A */
    printf("%X\n", setbits(0xFF00, 4, 4, 0x000A));   /* expect FFA0 */
    return 0;
}

unsigned setbits(unsigned x, int p, int n, unsigned y) {
    /* TODO */
    return x;
}
```

## Solution

```c
unsigned setbits(unsigned x, int p, int n, unsigned y) {
    unsigned ones_at_pos = ((1U << n) - 1) << p;     /* n ones at position p */
    unsigned y_in_pos    = (y & ((1U << n) - 1)) << p; /* y's bottom n bits, shifted */
    return (x & ~ones_at_pos) | y_in_pos;
}
```

```output
FFA0
```

## Notes from the author

- The `(1U << n) - 1` trick for "n ones" is so useful it's worth its own name in your head. Combined with `<< p`, it places a window of `n` bits anywhere.
- I masked `y` with `(1U << n) - 1` before shifting, just in case the caller passes a `y` with extra high bits set. The K&R prompt promises "rightmost n bits of y," so masking is defensive - but cheap.
- Watch out: `1U << n` where `n` equals the type's width is **undefined behaviour**. So this routine works for `n` strictly less than `sizeof(unsigned) * CHAR_BIT`. For `n == 32` on a 32-bit `unsigned`, you'd need a special case (return 0 mask, or use `~0U` instead of the formula).
