> Modify `entab` and `detab` (from earlier exercises in Ch 1) to accept a list of tab stops as arguments. With no arguments, use every 8 columns.

## Approach

Parse `argv` for integer tab stops. Build a function `next_tab(col, stops, n)` that returns the next stop ≥ `col + 1`. The rest of `entab`/`detab` is unchanged - they call this helper instead of computing `(col / 8 + 1) * 8`.

```c:starter
#include <stdio.h>
#include <stdlib.h>

#define MAXTABS 64
static int tabs[MAXTABS];
static int ntabs = 0;
static int default_step = 8;

/* return the column of the next stop strictly greater than `col` */
static int next_stop(int col) {
    if (ntabs == 0)
        return ((col / default_step) + 1) * default_step;
    /* find first tab > col */
    for (int i = 0; i < ntabs; ++i)
        if (tabs[i] > col)
            return tabs[i];
    /* past the last stop: keep stepping by default_step */
    int last = tabs[ntabs - 1];
    return last + ((col - last) / default_step + 1) * default_step;
}

static void detab_stream(void) {
    int c, col = 0;
    while ((c = getchar()) != EOF) {
        if (c == '\t') {
            int target = next_stop(col);
            while (col < target) { putchar(' '); ++col; }
        } else if (c == '\n') {
            putchar('\n'); col = 0;
        } else {
            putchar(c); ++col;
        }
    }
}

int main(int argc, char *argv[]) {
    for (int i = 1; i < argc && ntabs < MAXTABS; ++i)
        tabs[ntabs++] = atoi(argv[i]);
    detab_stream();
    return 0;
}
```

Usage:

```bash
$ printf 'a\tb\tc\n' | ./detab          # default every-8
a       b       c
$ printf 'a\tb\tc\n' | ./detab 4 10 16  # stops at columns 4, 10, 16
a   b     c
```

## Notes from the author

- Explicit tab stops are how legacy formatters (`groff`, `nroff`, `cat -t`) handle non-uniform columns. Modern editors mostly fix tabs at multiples of 4 or 8, but the flexible mechanism is useful for column-aligned tables.
- The `next_stop` helper centralises the "where's the next column" logic. Once you have that, both `entab` (collapse spaces back to tabs) and `detab` (expand) share the same column model.
- For sticking-to-defaults parameters, a single helper with sensible fallbacks beats forking the code path. `next_stop` handles both "no args" and "args" cleanly.
