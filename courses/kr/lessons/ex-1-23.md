> **Exercise 1-23.** Write a program to remove all comments from a C program. Don't forget to handle quoted strings and character constants properly. C comments do not nest.

## Approach

This is the first exercise that **requires a real state machine** — implicit "previous char" tracking is no longer enough. The state captures *where* in the source code we are:

- `CODE` — normal C code outside any string/comment
- `STRING` — inside a double-quoted string literal
- `CHAR` — inside a single-quoted character constant
- `COMMENT_LINE` — inside a `//` comment (until newline)
- `COMMENT_BLOCK` — inside a `/* ... */` comment

Transitions:

| From | On | To | Emit? |
|---|---|---|---|
| CODE | `"` | STRING | yes |
| CODE | `'` | CHAR | yes |
| CODE | `/` followed by `*` | COMMENT_BLOCK | no |
| CODE | `/` followed by `/` | COMMENT_LINE | no |
| STRING | `\`, then any | STRING | yes both |
| STRING | `"` | CODE | yes |
| CHAR | `\`, then any | CHAR | yes both |
| CHAR | `'` | CODE | yes |
| COMMENT_LINE | `\n` | CODE | yes the `\n` |
| COMMENT_BLOCK | `*` followed by `/` | CODE | no |

## Solution

```c
#include <stdio.h>

enum state { CODE, STRING, CHAR_LIT, LINE_CMT, BLOCK_CMT };

int main(void) {
    int  c;
    int  prev = 0;            /* previous char, for digraph detection */
    enum state st = CODE;

    while ((c = getchar()) != EOF) {
        switch (st) {
            case CODE:
                if (prev == '/' && c == '*') {
                    /* we already emitted '/', take it back */
                    fseek(stdout, -1, SEEK_CUR);  /* won't work on a pipe */
                    st = BLOCK_CMT;
                    prev = 0;
                    continue;
                }
                if (prev == '/' && c == '/') {
                    fseek(stdout, -1, SEEK_CUR);  /* same caveat */
                    st = LINE_CMT;
                    prev = 0;
                    continue;
                }
                putchar(c);
                if (c == '"')  st = STRING;
                if (c == '\'') st = CHAR_LIT;
                prev = c;
                break;

            case STRING:
                putchar(c);
                if (c == '\\') {           /* consume next char raw */
                    int n = getchar();
                    if (n != EOF) putchar(n);
                } else if (c == '"') {
                    st = CODE;
                }
                prev = 0;
                break;

            case CHAR_LIT:
                putchar(c);
                if (c == '\\') {
                    int n = getchar();
                    if (n != EOF) putchar(n);
                } else if (c == '\'') {
                    st = CODE;
                }
                prev = 0;
                break;

            case LINE_CMT:
                if (c == '\n') {
                    putchar('\n');
                    st = CODE;
                }
                prev = 0;
                break;

            case BLOCK_CMT:
                if (prev == '*' && c == '/') {
                    st = CODE;
                    prev = 0;
                } else {
                    prev = c;
                }
                break;
        }
    }
    return 0;
}
```

⚠️ **Caveat about the solution above:** the `fseek(stdout, -1, SEEK_CUR)` trick only works when stdout is a regular file, not a pipe or terminal. The honest fix is **lookahead**: buffer the `/` for one step and decide what to emit *after* peeking at the next character.

### Lookahead version (the right way)

```c
#include <stdio.h>

enum state { CODE, STRING, CHAR_LIT, LINE_CMT, BLOCK_CMT };

int main(void) {
    int  c, pending = 0;        /* pending = a '/' we're holding back */
    enum state st = CODE;

    while ((c = getchar()) != EOF) {
        if (st == CODE) {
            if (pending) {
                if (c == '*') { st = BLOCK_CMT; pending = 0; continue; }
                if (c == '/') { st = LINE_CMT;  pending = 0; continue; }
                putchar('/');
                pending = 0;
            }
            if (c == '/') {
                pending = 1;
            } else {
                putchar(c);
                if (c == '"')  st = STRING;
                if (c == '\'') st = CHAR_LIT;
            }
        } else if (st == STRING || st == CHAR_LIT) {
            putchar(c);
            if (c == '\\') {
                int n = getchar();
                if (n != EOF) putchar(n);
            } else if ((st == STRING && c == '"') ||
                       (st == CHAR_LIT && c == '\'')) {
                st = CODE;
            }
        } else if (st == LINE_CMT) {
            if (c == '\n') { putchar('\n'); st = CODE; }
        } else if (st == BLOCK_CMT) {
            if (c == '*') {
                int n = getchar();
                if (n == EOF) break;
                if (n == '/') st = CODE;
                /* otherwise keep eating; if 'n' is '*' we need to re-check
                   on the next iteration, which we do by pushing it back */
                if (n != '/' && n == '*') {
                    /* ungetc the second '*' so the outer loop sees it */
                    ungetc(n, stdin);
                }
            }
        }
    }

    if (pending) putchar('/');
    return 0;
}
```

## Notes from the author

- The `pending` variable is doing a **one-character lookahead**. Most parsers need exactly this — the literature calls it LL(1). If you ever need LL(2), it's time to consider a real parser generator (`bison`/`yacc`).
- I handled `\\` inside strings/chars to skip *any* escaped character (including `\"`, `\\`, `\n` literally typed as two chars). Without that, `"a/*not a comment*/b"` would be wrongly stripped.
- I deliberately left **trigraphs**, **raw string literals (C++11+)**, and **digraphs** out. K&R-era C doesn't have them and they would complicate the state machine.
- A real `cpp -P -fdirectives-only` does this and more. But coding the state machine by hand is exactly the kind of muscle you need for chapter 6 (parsers, abstract syntax) and any future language work.
