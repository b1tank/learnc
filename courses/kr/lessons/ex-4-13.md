> Write a recursive version of `reverse(s)` that reverses a string in place.

## Approach

Recursion mirrors the iterative two-pointer approach. The recursive structure: swap the outer pair (first and last), then recurse on the substring between them.

A clean way: pass the indices `i` and `j` (left and right) as parameters. The wrapper hides this.

```c:starter
#include <stdio.h>
#include <string.h>

void reverse_r(char s[], int i, int j);

void reverse(char s[]) {
    reverse_r(s, 0, (int)strlen(s) - 1);
}

void reverse_r(char s[], int i, int j) {
    if (i < j) {
        char t = s[i]; s[i] = s[j]; s[j] = t;
        reverse_r(s, i + 1, j - 1);
    }
}

int main(void) {
    char a[] = "hello";
    char b[] = "OpenAI";
    char c[] = "abcdef";
    char d[] = "x";
    char e[] = "";

    reverse(a); printf("[%s]\n", a);
    reverse(b); printf("[%s]\n", b);
    reverse(c); printf("[%s]\n", c);
    reverse(d); printf("[%s]\n", d);
    reverse(e); printf("[%s]\n", e);
    return 0;
}
```

```output
[olleh]
[IAnepO]
[fedcba]
[x]
[]
```

## Recursion depth

Each recursive call reduces the gap `j - i` by 2. The depth is `⌈n/2⌉` - so a 1 MB string would recurse ~500,000 levels and overflow the stack.

For real string-reversal you'd use the iterative two-pointer loop (`while (i < j) { swap; ++i; --j; }`). Recursion is the *exercise* here, not the production choice.

## Tail-call optimisation could save the day

The recursive call is the **last** thing the function does, so a smart compiler can convert it to a jump - no stack growth. Both GCC and Clang do this at `-O2`. To verify, compile with `gcc -O2 -S` and look at the generated assembly; you should see a jump back to the top instead of a `call`.

But the standard doesn't *guarantee* this. For deep recursion, write the loop.

## Notes from the author

- "Recursion + in-place mutation" is a slightly unusual combination. The pure-functional version would build a new reversed string and return it; the in-place version mutates the input. Both are valid; the in-place version saves memory.
- The wrapper-and-worker split (`reverse` calls `reverse_r`) is a clean way to hide implementation details. The public API takes one parameter; the internal recursive helper takes three. Users don't see the indices.
- Sometimes the best way to "see" a recursive algorithm is to draw the call tree on paper. For `reverse_r("abcdef", 0, 5)`: swap[0,5], call(1,4), swap[1,4], call(2,3), swap[2,3], call(3,2), return. Five swaps for six characters - exactly `⌈n/2⌉`.
