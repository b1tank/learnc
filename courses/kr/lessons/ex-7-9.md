> Functions like `isupper` can be implemented as **functions** (with overhead per call) or as **macros** that look up the answer in a 256-entry table. Show both versions and compare.

## Approach

The table:

```c
#define _U  0x01    /* upper */
#define _L  0x02    /* lower */
#define _D  0x04    /* digit */
#define _S  0x08    /* space-ish */
#define _P  0x10    /* punct */
/* ... more flags ... */

static const unsigned char ctype_table[256] = {
    /* 0x00..0x1F: control characters */
    [' ']  = _S,
    ['\t'] = _S,
    /* digits */
    ['0'] = _D, ['1'] = _D, /* ... ['9'] = _D */
    /* uppercase */
    ['A'] = _U, /* ... ['Z'] = _U */
    /* lowercase */
    ['a'] = _L, /* ... ['z'] = _L */
    /* etc. */
};

#define isupper_m(c) (ctype_table[(unsigned char)(c)] & _U)
```

The macro is just a table lookup + a bitwise AND - typically 2-3 instructions. A function call costs ~5-10 instructions (stack setup, return, function preamble).

```c:starter
#include <stdio.h>
#include <ctype.h>
#include <time.h>

#define _U 0x01
#define _L 0x02
#define _D 0x04

/* Build the table by initialising every byte we care about. */
static unsigned char tab[256];

static void build_table(void) {
    for (int c = 'A'; c <= 'Z'; ++c) tab[c] |= _U;
    for (int c = 'a'; c <= 'z'; ++c) tab[c] |= _L;
    for (int c = '0'; c <= '9'; ++c) tab[c] |= _D;
}

#define isupper_m(c) (tab[(unsigned char)(c)] & _U)
#define islower_m(c) (tab[(unsigned char)(c)] & _L)

int isupper_f(int c) { return tab[(unsigned char)c] & _U; }

int main(void) {
    build_table();

    const int N = 10000000;
    clock_t t1 = clock();
    int count = 0;
    for (int i = 0; i < N; ++i)
        count += isupper_m('A' + (i & 31));
    clock_t t2 = clock();
    printf("macro:   count=%d, time=%.3fs\n", count, (double)(t2-t1)/CLOCKS_PER_SEC);

    t1 = clock();
    count = 0;
    for (int i = 0; i < N; ++i)
        count += isupper_f('A' + (i & 31));
    t2 = clock();
    printf("function: count=%d, time=%.3fs\n", count, (double)(t2-t1)/CLOCKS_PER_SEC);

    return 0;
}
```

```output
macro:   count=...  time=0.020s
function: count=... time=0.080s
```

(Numbers vary by machine; the macro is consistently faster.)

## Why the table approach?

1. **Inline cost** - the expansion is open-coded; no function call.
2. **Branch-free** - the answer is a single arithmetic expression.
3. **One table for many predicates** - `isalpha`, `isdigit`, `isspace`, `isupper`, `islower`, all share the same table with different mask bits.

This is why real `<ctype.h>` implementations use this exact pattern - the macros expand to indexed table lookups. The "function" form (e.g. `(isupper)(c)`, parens to defeat the macro) exists for taking addresses and is implemented as a regular function call.

## Macro caveats

```c
#define isupper(c) (...)

isupper(*p++)   /* on glibc's macro: p is incremented TWICE! */
```

If the macro evaluates `c` more than once and `c` has side effects, you get a bug. The glibc macros are written carefully to evaluate `c` exactly once, but you can't always assume that. Modern compilers will often expand `static inline` functions just as well, with no side-effect risk:

```c
static inline int isupper_inline(int c) {
    return tab[(unsigned char)c] & _U;
}
```

This is the recommended modern form - fast as a macro, safe as a function.

## Try it

1. Add `_S` (space), `_P` (punct), `_X` (xdigit) bits and the corresponding lookups.
2. Print a 16x16 grid showing the flag bits for each byte. Visualise the table.
3. Benchmark `isupper_inline` (a `static inline` function) vs. the macro. They should be identical at `-O2`.

## Notes from the author

- "Macros that look like functions" was the K&R-era performance hack. Modern compilers inline `static inline` functions just as well, with proper scoping and no double-evaluation. Prefer `static inline`.
- The ctype table is the canonical example of a "lookup-table optimisation". Once you see the pattern (precompute → fast access), you'll apply it elsewhere: parse-state-transition tables, codecs, character-encoding mappings.
- POSIX locales add complexity: `isupper` returns true for accented Unicode letters in `en_US.UTF-8`. The 256-byte table works only for the C/POSIX locale; full Unicode requires bigger tables or table-driven property data (ICU's u_char_property).
