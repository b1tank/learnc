> Write `fopen` and `fclose` for a stripped-down version of `stdio` that supports only a fixed number of simultaneously-open files. Reuse slots in a static array.

## Approach

A small static array of `FILE *`-like structs. `fopen` finds an unused slot, opens the OS fd, fills the slot. `fclose` closes the fd and marks the slot unused.

```c:starter
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <fcntl.h>
#include <string.h>

#define BUFSIZE 1024
#define OPEN_MAX 16

typedef struct {
    int  fd;
    char buf[BUFSIZE];
    int  pos;
    int  end;
    int  flags;             /* bit 0: in use */
} mini_FILE;

static mini_FILE pool[OPEN_MAX];

mini_FILE *mini_fopen(const char *path, const char *mode) {
    int flags;
    if      (mode[0] == 'r') flags = O_RDONLY;
    else if (mode[0] == 'w') flags = O_WRONLY | O_CREAT | O_TRUNC;
    else if (mode[0] == 'a') flags = O_WRONLY | O_CREAT | O_APPEND;
    else return NULL;

    /* find a free slot */
    int idx = -1;
    for (int i = 0; i < OPEN_MAX; ++i) {
        if ((pool[i].flags & 1) == 0) { idx = i; break; }
    }
    if (idx < 0) {
        fprintf(stderr, "mini_fopen: too many open files\n");
        return NULL;
    }

    int fd = open(path, flags, 0644);
    if (fd < 0) return NULL;

    mini_FILE *fp = &pool[idx];
    fp->fd     = fd;
    fp->pos    = fp->end = 0;
    fp->flags  = 1;
    return fp;
}

int mini_fclose(mini_FILE *fp) {
    if (!fp || (fp->flags & 1) == 0) return -1;
    int rc = close(fp->fd);
    fp->flags = 0;
    return rc;
}

int mini_getc(mini_FILE *fp) {
    if (fp->pos >= fp->end) {
        ssize_t n = read(fp->fd, fp->buf, BUFSIZE);
        if (n <= 0) return EOF;
        fp->pos = 0;
        fp->end = (int)n;
    }
    return (unsigned char)fp->buf[fp->pos++];
}

int main(void) {
    mini_FILE *fp = mini_fopen("/etc/hostname", "r");
    if (!fp) { perror("hostname"); return 1; }
    int c, n = 0;
    while ((c = mini_getc(fp)) != EOF) { putchar(c); ++n; }
    mini_fclose(fp);
    printf("\nread %d bytes\n", n);
    return 0;
}
```

```output
yummy.local
read 12 bytes
```

(Output depends on `/etc/hostname`.)

## Why a fixed-size pool

- Real `stdio.h` has `FOPEN_MAX` (default 16 on many systems) — the max number of files you can have open simultaneously through the C stdio interface.
- Static allocation means no `malloc` overhead per open. Predictable memory cost.
- Reuse means a long-running program with thousands of "open, read, close" cycles never grows.

## Detection of "all slots in use"

Linear scan in the pool — `O(n)` but `n ≤ 16`, so fine. For bigger pools, a free-list of indices is faster.

## Try it

1. Add `mini_fwrite`, `mini_fread` so the pool supports writing too.
2. Track the maximum slot index ever used; print at exit.
3. What happens if you `mini_fclose` twice on the same pointer? Add detection.

## Notes from the author

- The "pool of resources" pattern recurs everywhere: thread pools, connection pools, page caches. Fixed size, find-by-scan, recycle on release.
- Real stdio's `fopen` does `malloc` for the `FILE` struct on most platforms. The fixed pool is a teaching simplification. The fixed pool is also how kernels manage some scarce resources (e.g. inode tables).
- "Slot reuse" can hide bugs: a stale pointer from a closed slot now points to a reopened one. Generation counts (`fp->generation++` on close) and per-slot tokens fix this in serious code.
