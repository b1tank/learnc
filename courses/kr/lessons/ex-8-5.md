> The §8.6 directory walker uses `stat`, which follows symlinks. Modify it to use `lstat` (no follow), report symlink targets, and avoid infinite loops on symlink cycles.

## Approach

Two changes:

1. **`lstat` instead of `stat`** - tells us about the symlink itself, not its target.
2. **Cycle detection** - keep a set of visited inode/device pairs; skip directories we've seen.

```c:starter
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <dirent.h>
#include <sys/stat.h>
#include <unistd.h>

#define MAX_VISITED 1024

struct id { dev_t dev; ino_t ino; };
static struct id visited[MAX_VISITED];
static int n_visited = 0;

static int seen(dev_t d, ino_t i) {
    for (int k = 0; k < n_visited; ++k)
        if (visited[k].dev == d && visited[k].ino == i)
            return 1;
    if (n_visited < MAX_VISITED) {
        visited[n_visited++] = (struct id){ d, i };
    }
    return 0;
}

static void walk(const char *path) {
    struct stat st;
    if (lstat(path, &st) < 0) { perror(path); return; }

    if (S_ISLNK(st.st_mode)) {
        char target[1024];
        ssize_t n = readlink(path, target, sizeof target - 1);
        if (n > 0) target[n] = '\0';
        else       strcpy(target, "?");
        printf("L %s -> %s\n", path, target);
        return;            /* don't follow */
    }

    if (S_ISREG(st.st_mode)) {
        printf("- %8lld %s\n", (long long)st.st_size, path);
        return;
    }
    if (!S_ISDIR(st.st_mode)) {
        printf("? %s\n", path);
        return;
    }

    /* directory - check for cycles */
    if (seen(st.st_dev, st.st_ino)) {
        printf("(cycle detected, skipping %s)\n", path);
        return;
    }
    printf("d %s\n", path);

    DIR *dp = opendir(path);
    if (!dp) { perror(path); return; }
    struct dirent *de;
    while ((de = readdir(dp))) {
        if (strcmp(de->d_name, ".") == 0 || strcmp(de->d_name, "..") == 0)
            continue;
        char child[1024];
        snprintf(child, sizeof child, "%s/%s", path, de->d_name);
        walk(child);
    }
    closedir(dp);
}

int main(int argc, char *argv[]) {
    walk(argc > 1 ? argv[1] : ".");
    return 0;
}
```


## Why dev+inode

A file is uniquely identified by `(st_dev, st_ino)` - the device and the inode number. Two paths can resolve to the same file via hard links or bind mounts; their `(dev, ino)` is the same.

This is the standard cycle-detection key for filesystem walks. Real `find(1)` does this; it's the `find -L` (logical follow with cycle detect) implementation.

## `readlink` semantics

```c
ssize_t readlink(const char *path, char *buf, size_t n);
```

Reads the *target* of a symlink (NOT the file it points to). Does **NOT** null-terminate the buffer - you have to.

If `buf` is too small, the result is truncated silently. Use a buffer at least `PATH_MAX` long, or call repeatedly with growing buffers.

3. Build a "find -type X" filter.

## Notes from the author

- `lstat` vs `stat`: the former *describes* a symlink; the latter *follows* it to describe the target. Always think about which you want - `find`, `du`, `ls -l` use different ones for different reasons.
- Cycle detection by `(dev, ino)` is the algorithm `find` uses. Without it, a symlink `dir/loop -> ..` causes infinite descent.
- On networked filesystems (NFS), inode numbers can collide across mounts - `(dev, ino)` is the correct identifier. Don't use just `ino`.
