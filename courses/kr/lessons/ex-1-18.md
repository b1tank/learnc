> **Exercise 1-18.** Write a program that removes trailing blanks and tabs from each input line, and that deletes entirely blank lines.

## Approach

Read each line, then walk *backwards* from the end skipping spaces and tabs. If what remains is empty, drop the line; otherwise print the trimmed version with a single trailing newline.

```c:starter
#include <stdio.h>

#define MAXLINE 1000

int  read_line(char s[], int maxsize);
int  rstrip(char s[]);     /* returns new length */

int main(void) {
    /* TODO */
    return 0;
}
```

## Solution

```c
#include <stdio.h>

#define MAXLINE 1000

int read_line(char s[], int maxsize) {
    int c, i = 0;
    while (i < maxsize - 1 && (c = getchar()) != EOF && c != '\n')
        s[i++] = c;
    if (c == '\n') s[i++] = '\n';
    s[i] = '\0';
    return i;
}

/* Strip trailing spaces, tabs, and the terminal newline.
   Returns the new length. */
int rstrip(char s[], int len) {
    while (len > 0) {
        char c = s[len - 1];
        if (c == ' ' || c == '\t' || c == '\n') --len;
        else break;
    }
    s[len] = '\0';
    return len;
}

int main(void) {
    char line[MAXLINE];
    int  len;

    while ((len = read_line(line, MAXLINE)) > 0) {
        len = rstrip(line, len);
        if (len > 0)
            printf("%s\n", line);
    }
    return 0;
}
```

## Notes from the author

- The trick is to do trimming *and* blank-line detection in one shot. After `rstrip`, "blank line" reduces to "length is zero." No special case needed.
- I made `rstrip` mutate the buffer and return the new length. The alternative is to return a pointer; either is fine in K&R-era C. The length-based interface plays well with our `read_line` contract.
- A common bug: forgetting to strip the trailing `\n` before the trailing space test. If the line is `"hello   \n"`, peeling the `\n` first lets the space loop do its job. My loop handles all three (`\n`, space, tab) in one predicate, which is simpler.
- Real-world cousin: `sed -e 's/[ \t]*$//' -e '/^$/d'`. Programs that ingest user-typed input usually trim like this before parsing.
