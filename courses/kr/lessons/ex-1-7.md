> **Exercise 1-7.** Write a program that prints the numeric value of `EOF`.

## Approach

`EOF` is a `#define` from `<stdio.h>`. The standard requires it to be a negative `int` distinct from every byte (0–255), but doesn't pin down *which* negative integer. In practice every implementation uses `-1`. Print it and see.

```c:starter
#include <stdio.h>

int main(void) {
    /* TODO */
    return 0;
}
```

## Solution

```c
#include <stdio.h>

int main(void) {
    printf("EOF = %d\n", EOF);
    return 0;
}
```

```output
EOF = -1
```

## Notes from the author

- One-liner. The point is the *concept*: `EOF` is a macro whose value the implementation chooses, not a magic language constant. You'd never write `if (c == -1)` in real code because some exotic platform could `#define EOF -2` and your check would silently fail. Always compare against `EOF` by name.
- Why negative? Because `getchar` returns `unsigned char` cast to `int` for actual bytes (0–255), so any negative value is unambiguously "not a byte." Same convention used by `getc`, `fgetc`, `ungetc`.
- A more interesting variant: print `EOF` as both a signed and unsigned integer — the bit pattern is the same, but the displayed value differs because `unsigned int` interprets the high bit as magnitude rather than sign. We'll meet this in §2.7 (type conversions).
