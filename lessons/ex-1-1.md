> **Exercise 1-1.** Run the hello-world program from §1.1 on your system. Now intentionally remove pieces — the `#include`, the semicolon, the quotes, the `int`, the `return` — one at a time and read each error message carefully. Error messages are a vocabulary; the sooner you can map them to causes the faster every later bug becomes.

## Approach

This is a reading exercise, not a writing one. The point is to build a *catalogue* of common error messages so when you hit them in real code you recognise the shape immediately. Below is the working program; the experiments are listed under **Try it**. Click run after each modification and write down the first line of the compiler's complaint.

```c:starter
#include <stdio.h>

int main(void) {
    printf("hello, world\n");
    return 0;
}
```

```output
hello, world
```

## Try it

Edit the starter, run, observe, then revert. Repeat for each row.

| Change | What you'll typically see |
| --- | --- |
| Delete `#include <stdio.h>` | *implicit declaration of function `printf`* — the compiler doesn't know `printf` exists. |
| Delete the `;` after `printf(...)` | *expected `;` before `return`* — parser is lost. |
| Delete one of the `"` quotes around `hello, world\n` | *missing terminating `"` character* and a cascade of follow-up errors. |
| Change `int main` to `void main` | *return type of `main` is not `int`* (warning, sometimes error). |
| Delete the `return 0;` | In C99+ the compiler will silently insert one. In C89 you get a warning. |
| Replace `\n` with `\m` | *unknown escape sequence `\m`* — warning, program still compiles, output has a stray `m`. |
| Rename `main` to `Main` | *undefined reference to `main`* at link time — the linker can't find the entry point. |

## Notes from the author

- The point isn't to remember exactly which compiler emits which message — it's to learn that there's a small zoo of recurring complaints. By the time you've broken hello-world ten ways, you've seen most of them.
- The "implicit declaration" error from removing `#include <stdio.h>` is one of the most common errors C99 era. In C11+ it's escalated to a real error rather than a warning, which is good — historical C would have produced linkable code that crashes at run time.
- The K&R-flavoured exercise 1-1 asks specifically about hello-world, but the *pattern* — read errors, build vocabulary — applies to every language. When you teach this to someone else, do it live: break the code, read the message, fix it.
