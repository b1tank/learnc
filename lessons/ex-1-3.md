> **Exercise 1-3.** Modify the Fahrenheit-to-Celsius converter from §1.2 so it prints a column heading above the table — `Fahr` over the first column, `Celsius` over the second.

## Approach

Drop a `printf` above the loop. The only nuance is column alignment: the heading text widths must match the data widths in the existing `printf` format so the columns don't drift.

```c:starter
#include <stdio.h>

int main(void) {
    int fahr;

    /* TODO: print heading row */
    for (fahr = 0; fahr <= 300; fahr += 20)
        printf("%3d %6.1f\n", fahr, (5.0/9.0) * (fahr - 32));

    return 0;
}
```

## Solution

```c
#include <stdio.h>

int main(void) {
    int fahr;

    printf("%3s %6s\n", "Fahr", "Celsius");
    printf("%3s %6s\n", "----", "-------");

    for (fahr = 0; fahr <= 300; fahr += 20)
        printf("%3d %6.1f\n", fahr, (5.0/9.0) * (fahr - 32));

    return 0;
}
```

## Notes from the author

- `%s` accepts any width specifier, so reusing `%3s %6s` keeps the heading aligned with the body's `%3d %6.1f`. If you change the data widths later, change the heading too — a tiny example of the "single source of truth" rule.
- The "underline" row is optional but makes the boundary unmistakable. Real-world TTY tables almost always have one.
- A more disciplined version uses `#define` to make the widths symbolic — `#define FW 3` / `#define CW 6` and then `printf("%*s %*s\n", FW, "Fahr", CW, "Celsius")`. `%*s` reads the width from the next argument. Worth showing once the §1.4 lesson lands.
