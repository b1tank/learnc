> Write a program that compares two files line by line and prints the first line on which they differ.

## Approach

Open both files. Read a line from each. If both EOF together, the files are equal. If one EOFs first, that's a difference. If lines differ, print and stop.

```c:starter
#include <stdio.h>
#include <string.h>

int main(int argc, char *argv[]) {
    if (argc != 3) {
        fprintf(stderr, "usage: %s FILE1 FILE2\n", argv[0]);
        return 2;
    }
    FILE *a = fopen(argv[1], "r");
    FILE *b = fopen(argv[2], "r");
    if (!a) { perror(argv[1]); return 2; }
    if (!b) { perror(argv[2]); fclose(a); return 2; }

    char la[4096], lb[4096];
    int line = 0;
    for (;;) {
        ++line;
        char *ra = fgets(la, sizeof la, a);
        char *rb = fgets(lb, sizeof lb, b);
        if (!ra && !rb) {
            printf("files are identical\n");
            fclose(a); fclose(b);
            return 0;
        }
        if (!ra) {
            printf("line %d: %s ends; %s has: %s", line, argv[1], argv[2], lb);
            fclose(a); fclose(b);
            return 1;
        }
        if (!rb) {
            printf("line %d: %s ends; %s has: %s", line, argv[2], argv[1], la);
            fclose(a); fclose(b);
            return 1;
        }
        if (strcmp(la, lb) != 0) {
            printf("line %d differs:\n  %s: %s", line, argv[1], la);
            printf("  %s: %s", argv[2], lb);
            fclose(a); fclose(b);
            return 1;
        }
    }
}
```

Test:

```bash
$ printf 'apple\nbanana\ncherry\n' > a.txt
$ printf 'apple\nbanana\nCHERRY\n' > b.txt
$ ./diff a.txt b.txt
line 3 differs:
  a.txt: cherry
  b.txt: CHERRY
```

## Edge cases

- One file ends before the other — handled by checking `ra`/`rb` separately.
- Lines longer than 4096 — `fgets` reads only a prefix. Two long lines that match in the prefix but differ in the suffix would be wrongly called equal. Fix: use `getline` (POSIX) to read full lines regardless of length.
- Binary differences mid-line — `strcmp` stops at the first `\0`, but `fgets` reads bytes literally including embedded nulls. Use `memcmp` for byte-exact comparison.

## Try it

1. Print ALL differing lines, not just the first.
2. Add a `-q` flag that prints nothing, returns 0 if equal and 1 if not.
3. Replace `fgets` with `getline` to handle unlimited line lengths.

## Notes from the author

- This is a trivial "diff" — real `diff(1)` runs a longest-common-subsequence algorithm to find minimal edits. That's a 200-line Hunt–McIlroy or Myers algorithm; out of scope here.
- Reporting "first difference" is enough for many use cases (e.g. CI test output comparison). For richer reports, build on `diff` or use `git diff`.
- Always return non-zero on differences so callers can scripted-check. Returning 2 for usage errors and 1 for "differences found" is the Unix convention.
