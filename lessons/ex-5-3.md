> Write a pointer-walking version of `strcat(s, t)` that appends `t` to the end of `s`.

## Approach

Two phases:

1. Walk `s` until its null terminator. After the loop, the pointer points to the `'\0'`.
2. Copy `t` (including its terminator) into the place starting at that null.

```c:starter
#include <stdio.h>

void my_strcat(char *s, const char *t);

int main(void) {
    char buf[64] = "hello ";
    my_strcat(buf, "pointer ");
    my_strcat(buf, "world");
    printf("[%s]\n", buf);
    return 0;
}

void my_strcat(char *s, const char *t) {
    while (*s)                /* find end of s */
        ++s;
    while ((*s++ = *t++) != '\0')   /* copy t (with terminator) */
        ;
}
```

```output
[hello pointer world]
```

## Notes from the author

- The two-phase split (find end, then append) is one of those C idioms that hides in plain sight. The first loop's "advance until 0" *also* leaves the pointer in the right place for the second loop — no extra index math.
- `strcat` is famously unsafe — there's no bound on how much you write to `s`. Use `strncat` or just `snprintf` for any user-supplied input. Modern security advisories list `strcat` as "never use".
- The walking pattern repeats in `strcpy`, `strchr`, `strstr`, `strpbrk` — every `<string.h>` function. Internalising it pays off across the chapter.
