> **Exercise 1-13.** Print a histogram showing the distribution of word lengths in the input. Try both a horizontal and a vertical version.

## Approach

State machine again, this time tracking the *current word's length*. When the word ends, bump the count for that length and reset.

A small array indexed by word length holds the buckets. Decide an upper bound (say 20) and lump anything longer into the last bucket.

## Solution — horizontal bars

```c
#include <stdio.h>

#define MAXLEN  20      /* longest tracked word length */
#define OUT     0
#define IN      1

int main(void) {
    int c, state = OUT, wlen = 0;
    int hist[MAXLEN + 1] = {0};    /* hist[k] = count of words of length k */
    int over = 0;                   /* count of words longer than MAXLEN */

    while ((c = getchar()) != EOF) {
        if (c == ' ' || c == '\t' || c == '\n') {
            if (state == IN) {
                if (wlen <= MAXLEN) ++hist[wlen];
                else                ++over;
                wlen = 0;
                state = OUT;
            }
        } else {
            ++wlen;
            state = IN;
        }
    }
    if (state == IN) {                 /* input ended mid-word */
        if (wlen <= MAXLEN) ++hist[wlen];
        else                ++over;
    }

    for (int len = 1; len <= MAXLEN; ++len) {
        printf("%3d | ", len);
        for (int j = 0; j < hist[len]; ++j) putchar('#');
        printf(" %d\n", hist[len]);
    }
    if (over > 0)
        printf(" >%d | (%d longer)\n", MAXLEN, over);

    return 0;
}
```

## Notes from the author

- The two natural orientations — bars going *across* (one row per bucket) vs *up* (one column per bucket) — exercise different formatting muscles. The horizontal version above is the easier one. Vertical bars need to know the max bucket height before printing, so you make a second pass.
- The end-of-input check (`if (state == IN)` after the loop) is essential for inputs without a trailing newline. I called this out in ex-1-12 too; it's the same trap.
- Stretch: instead of fixed-character `#` bars, *scale* the bars so the longest one fits in 60 columns. That's a real-world plotting concern — the data and the canvas have separate scales.
- The §1.5 word counter is a degenerate case of this histogram: count of all words = sum of all buckets.
