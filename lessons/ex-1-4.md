> **Exercise 1-4.** Write the inverse table: given Celsius values across a useful range, print the corresponding Fahrenheit values.

## Approach

Same loop skeleton as §1.2 / §1.3, different formula. The relation is `F = C × 9/5 + 32`. Pick a range that brackets the temperatures humans care about — say `-40` to `120` in steps of `10`.

The interesting line is the formula: like before, force floating-point arithmetic by writing `9.0/5.0` (or `9.0/5`) so the division isn't integer.

```c:starter
#include <stdio.h>

int main(void) {
    /* TODO */
    return 0;
}
```

## Solution

```c
#include <stdio.h>

int main(void) {
    int cel;

    printf("%4s %6s\n", "Cel", "Fahr");
    printf("%4s %6s\n", "----", "------");

    for (cel = -40; cel <= 120; cel += 10)
        printf("%4d %6.1f\n", cel, (9.0/5.0) * cel + 32);

    return 0;
}
```

## Notes from the author

- The crossover at `-40` is a famous trivia point: Celsius and Fahrenheit agree exactly at `-40°`. The output confirms it.
- Note the formula uses `(9.0/5.0) * cel` rather than `9 * cel / 5`. Both compute the same answer for clean integer Celsius inputs, but the first form makes the floating-point promotion explicit and survives more readers' code review. The latter form would be integer-divided if you ever moved the parentheses.
- A nice extension: print both tables side by side in a single program. The shape becomes `for (int x = LO; x <= HI; x += STEP)` driving two `printf` calls — a tiny exercise in functional decomposition you'll formalise in §1.7.
