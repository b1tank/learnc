> Use the recursive `print_digits` idea from §4.10 to write a recursive `itoa`. The challenge: the function must produce a *string* (in a buffer), not directly print to stdout.

## Approach

Same recursive structure as `print_digits`, but instead of `putchar`, we write into a buffer. The trick is that each recursive call needs to know *where* in the buffer to write — pass an index by pointer, or return the next-write position.

```c:starter
#include <stdio.h>

void itoa_r(int n, char s[], int *index);

void itoa(int n, char s[]) {
    int i = 0;
    if (n < 0) {
        s[i++] = '-';
        n = -n;        /* leaves INT_MIN as known bug */
    }
    itoa_r(n, s, &i);
    s[i] = '\0';
}

void itoa_r(int n, char s[], int *index) {
    if (n / 10 != 0)
        itoa_r(n / 10, s, index);
    s[(*index)++] = (char)(n % 10 + '0');
}

int main(void) {
    char buf[32];
    itoa(0,     buf);  printf("[%s]\n", buf);
    itoa(7,     buf);  printf("[%s]\n", buf);
    itoa(12345, buf);  printf("[%s]\n", buf);
    itoa(-42,   buf);  printf("[%s]\n", buf);
    return 0;
}
```

```output
[0]
[7]
[12345]
[-42]
```

## How it works

`itoa_r` is the recursive worker. On each call:

1. If `n` has more than one digit, recurse on `n / 10` first. This emits the higher-order digits.
2. Then write `n % 10` (the current low-order digit) at the current index.

The recursion does the reverse-the-digit-order job for free. The wrapper `itoa` handles the sign and the null terminator — those are *one-time* tasks that don't belong in the recursive worker.

## Why the index is a pointer

Each recursive call must advance the *shared* write position. If we passed `index` by value, every call would write to its own copy and only one digit would survive. The pointer makes "where to write next" persistent across the recursion.

Alternative: return the next-write index from the function, threading it through every call:

```c
int itoa_r(int n, char s[], int i) {
    if (n / 10 != 0)
        i = itoa_r(n / 10, s, i);
    s[i++] = (char)(n % 10 + '0');
    return i;
}
```

This is more "functional" (no shared mutation) but slightly more verbose. Both are fine; pick whichever feels more natural.

## Notes from the author

- Passing an "out-parameter index" by pointer is the C idiom for "this recursive function appends to a growing buffer". You'll see it in tree-flattening, JSON serialization, ASCII-art renderers.
- The base case is implicit: when `n / 10 == 0`, we skip the recursive call but still emit the single remaining digit. So `n == 0` correctly produces `"0"` — no special case needed.
- Modern C: prefer `snprintf(buf, len, "%d", n)`. It's bounded by `len`, correctly handles INT_MIN, and the standard library has had decades to optimise it. The point of writing it yourself is to internalise how digit-extraction recursion works.
