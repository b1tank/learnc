> **Exercise 1-20.** Write a program `detab` that replaces tabs in the input with the proper number of blanks to advance to the next tab stop. Assume a fixed tab stop every *n* columns.

## Approach

Maintain a column counter. On a regular character, emit it and advance the column. On a tab, emit blanks until the column is a multiple of *n*. On a newline, reset the column.

```c:starter
#include <stdio.h>

#define TABSTOP 8

int main(void) {
    /* TODO: replace tabs with blanks to next tab stop */
    return 0;
}
```

## Solution

```c
#include <stdio.h>

#define TABSTOP 8

int main(void) {
    int c, col = 0;

    while ((c = getchar()) != EOF) {
        if (c == '\t') {
            int spaces = TABSTOP - (col % TABSTOP);
            for (int i = 0; i < spaces; ++i) {
                putchar(' ');
                ++col;
            }
        } else if (c == '\n') {
            putchar(c);
            col = 0;
        } else {
            putchar(c);
            ++col;
        }
    }
    return 0;
}
```

```output
```

Input (`\t` is a literal tab): `abc\tdef\n`. With `TABSTOP = 8`: `abc` is 3 chars, the tab fills columns 3..7 (5 spaces) to reach column 8, then `def` is appended. Output: `abc     def`.

## Notes from the author

- The key formula is `spaces = TABSTOP - (col % TABSTOP)`. It always gives a positive number between 1 and TABSTOP — including the case where you're already on a tab stop boundary (`col % TABSTOP == 0`), in which case a tab inserts a full TABSTOP-wide gap. That matches Vim, GNU `expand`, and basically every other detab tool.
- I tracked column as `int`; on extremely long lines (>2 billion chars) this would overflow, but that's not a realistic concern for plain text.
- A common bug is to reset `col` on `\r` as well. Old Mac line endings and Windows CRLF can confuse a column tracker. For our purposes (Unix), only `\n` matters.
- Stretch: accept TABSTOP from `argv[1]` (chapter 5) so users can run `detab 4`. Also: support *non-uniform* tab stops (a list of columns) — that's how a real `expand -t 4,8,12` works.
