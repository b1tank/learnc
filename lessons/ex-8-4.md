> Add an `fseek(fp, offset, whence)` to your mini stdio. The tricky part: invalidate the buffer correctly so reads after the seek see the right data.

## Approach

After `fseek`:

1. Call `lseek` on the underlying fd to move the OS-level offset.
2. **Mark the buffer empty** so the next `getc` triggers a refill from the new position.

If you forget step 2, `getc` will continue returning bytes from the now-stale buffer, with no relationship to the new file position.

```c:starter
#include <stdio.h>
#include <unistd.h>
#include <fcntl.h>
#include <stdlib.h>

#define BUFSIZE 1024

typedef struct {
    int  fd;
    char buf[BUFSIZE];
    int  pos;
    int  end;
} mini_FILE;

static mini_FILE *mini_fopen(const char *path) {
    int fd = open(path, O_RDONLY);
    if (fd < 0) return NULL;
    mini_FILE *fp = calloc(1, sizeof *fp);
    fp->fd = fd;
    return fp;
}

static int mini_getc(mini_FILE *fp) {
    if (fp->pos >= fp->end) {
        ssize_t n = read(fp->fd, fp->buf, BUFSIZE);
        if (n <= 0) return EOF;
        fp->pos = 0;
        fp->end = (int)n;
    }
    return (unsigned char)fp->buf[fp->pos++];
}

static int mini_fseek(mini_FILE *fp, long offset, int whence) {
    off_t got = lseek(fp->fd, offset, whence);
    if (got < 0) return -1;
    fp->pos = fp->end = 0;     /* invalidate buffer */
    return 0;
}

static long mini_ftell(mini_FILE *fp) {
    off_t kpos = lseek(fp->fd, 0, SEEK_CUR);
    if (kpos < 0) return -1;
    /* kpos is where the kernel thinks we are; the buffer holds (end-pos) bytes
       NOT YET consumed by the user. The user's logical position is kpos - (end-pos). */
    return (long)kpos - (fp->end - fp->pos);
}

int main(int argc, char *argv[]) {
    const char *path = (argc > 1) ? argv[1] : "/etc/services";
    mini_FILE *fp = mini_fopen(path);
    if (!fp) { perror(path); return 1; }

    /* read first 10 bytes */
    for (int i = 0; i < 10; ++i) {
        int c = mini_getc(fp);
        if (c == EOF) break;
        putchar(c);
    }
    printf("\npos = %ld\n", mini_ftell(fp));

    /* jump to byte 100 */
    mini_fseek(fp, 100, SEEK_SET);
    printf("after seek to 100, pos = %ld\n", mini_ftell(fp));

    /* read 10 more */
    for (int i = 0; i < 10; ++i) {
        int c = mini_getc(fp);
        if (c == EOF) break;
        putchar(c);
    }
    putchar('\n');

    return 0;
}
```

```output
(depends on file)
```

## The `ftell` subtlety

When the user has consumed `pos` bytes out of `end` buffered bytes, the kernel offset is `end` bytes past the **logical** position. So:

```
logical_pos = kernel_pos - (end - pos)
```

Real `ftell` does this calculation. Without it, you'd return positions that are 1–4096 bytes too far.

## Try it

1. Test that "seek-then-read" returns the same bytes as "read-from-the-start-and-skip".
2. Add a `mini_rewind` that's just `mini_fseek(fp, 0, SEEK_SET)`.
3. Add `mini_fseek` for write streams. The buffer might contain unwritten data — flush before seeking!

## Notes from the author

- The "invalidate the buffer on seek" rule is the most common stdio bug to write. Anything that changes the underlying file position from outside the stdio layer breaks the cached buffer.
- For write streams, flush before seek; for read streams, discard the buffer. For `r+` (both directions), both rules apply — and you have to track which direction the buffer is currently used for.
- `fflush(fp)` on an input stream is technically undefined in pre-C23 (some libraries discard the buffer; some don't). Use `fseek(fp, 0, SEEK_CUR)` for a portable no-op-with-buffer-discard.
