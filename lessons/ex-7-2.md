> Write a program that prints arbitrary input "sensibly" — non-printing characters are shown in octal or hex, and lines are wrapped if too long. Useful for inspecting binary files.

## Approach

For each input byte, decide: is it printable? If yes, emit it as-is. If no, emit `\xNN` or `\NNN` (octal). Also track output column; insert a line break and continuation when the column gets near the terminal width.

```c:starter
#include <stdio.h>
#include <ctype.h>

#define MAXCOL 76

int main(void) {
    int c, col = 0;

    while ((c = getchar()) != EOF) {
        if (col >= MAXCOL - 4) {     /* room for an octal escape */
            putchar('\\'); putchar('\n');
            col = 0;
        }
        if (isprint(c)) {
            putchar(c);
            ++col;
        } else if (c == '\n') {
            putchar('\n');
            col = 0;
        } else if (c == '\t') {
            fputs("\\t", stdout);
            col += 2;
        } else if (c == '\\') {
            fputs("\\\\", stdout);
            col += 2;
        } else {
            /* fall back to octal */
            col += printf("\\%03o", (unsigned char)c);
        }
    }
    if (col > 0) putchar('\n');
    return 0;
}
```

```output
(feed any binary file)
```

Run on a small binary:

```bash
$ printf 'hello\0world\x1b[31m' | ./safe
hello\000world\033[31m
```

## Why the wrap?

Long lines are painful to view in terminals — they wrap, you can't see structure. By inserting a `\` + newline at column 72 or so, the output stays scannable while remaining unambiguous (the trailing `\` indicates continuation).

The C compiler reads `\` + newline the same way — it's the **line continuation** convention. So you could even paste the output back as C source.

## Variants of escape style

| Style    | Looks like      | Used by          |
|----------|------------------|------------------|
| `\NNN`   | `\033`           | C string literals |
| `\xNN`   | `\x1b`           | C string literals (C99+) |
| `^X`     | `^[`             | `cat -v`, terminals |
| `<XX>`   | `<1b>`           | some dump tools   |

The textbook calls for octal; modern tools more often use hex. Pick one and be consistent.

## Try it

1. Add a `-x` flag that switches to hex (`\xNN`) instead of octal.
2. Make the column width come from the `COLUMNS` environment variable (`getenv`).
3. Add a "hex dump" mode: 16 bytes per line, with offset prefix.

## Notes from the author

- "Make binary safe to view" is the job of tools like `cat -v`, `od -c`, `xxd`. Building one from scratch is a great exercise in I/O + escapes.
- The line-continuation trick (`\\` + newline) preserves the property that the output, if reparsed as C, gives back the same string. That's a beautiful invariant.
- For *truly* binary data (PDFs, executables), prefer `xxd` or `od` — they show offsets and aligned hex. The escape style is best for "mostly text with a few stray bytes".
