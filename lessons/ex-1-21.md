> **Exercise 1-21.** Write a program `entab` that replaces strings of blanks by the minimum number of tabs and blanks to achieve the same spacing. Use the same tab stop as `detab`.

## Approach

Read into a buffer (line at a time) and walk it column-by-column. Whenever you find a run of one or more blanks, decide:

- If the run crosses one or more tab stop boundaries, emit one tab for each boundary crossed and then emit the leftover blanks.
- If it doesn't cross a tab stop, leave it as blanks.

The cleanest implementation buffers the current run of blanks until you hit a non-blank (or the column reaches a tab stop), then decides how to flush.

## Solution

```c
#include <stdio.h>

#define TABSTOP 8

int main(void) {
    int c, col = 0, blanks = 0, start_col = 0;

    while ((c = getchar()) != EOF) {
        if (c == ' ') {
            if (blanks == 0) start_col = col;
            ++blanks;
            ++col;
            /* if we've just reached a tab stop while collecting blanks,
               we may be able to emit a tab now */
            if (col % TABSTOP == 0) {
                /* emit one tab if the run started before this stop */
                if (start_col / TABSTOP != col / TABSTOP) {
                    putchar('\t');
                    /* the tab consumes the whole run up to this stop */
                    blanks = 0;
                    start_col = col;
                }
            }
        } else {
            /* flush any leftover blanks as literal spaces */
            for (int i = 0; i < blanks; ++i) putchar(' ');
            blanks = 0;

            putchar(c);
            if (c == '\n') {
                col = 0;
            } else {
                ++col;
            }
            start_col = col;
        }
    }
    /* flush trailing blanks if input ended without a newline */
    for (int i = 0; i < blanks; ++i) putchar(' ');
    return 0;
}
```

Input: `abc          def` (12 spaces between `abc` and `def`, `TABSTOP = 8`). Columns: `abc` ends at col 3; the 12-blank run pushes col through 8 (tab boundary) to 15; then `def`. Output: `abc\t\tdef` (two tabs — one to reach col 8, one to reach col 16 — then `def` starts at col 16... wait, the run only reaches col 15. So the first 5 blanks become `\t` (reaching col 8), then the remaining 7 blanks fill cols 8..14 — still 7 blanks because they don't reach the next tab stop). So output: `abc\t       def`.

## Notes from the author

- `entab` is genuinely tricky, especially compared to `detab`. The asymmetry comes from "minimum number of tabs and blanks": you have to be careful not to insert a tab when a single space would have moved you the same distance.
- The trick I used is to flush at *tab-stop boundaries* rather than at the end of the blank run. That makes the bookkeeping easier — every time `col % TABSTOP == 0` and the run started before the current stop, swallow it as one tab.
- A subtle point: this program is **not safe inside string literals**. `printf("hello   world")` would have its internal blanks compacted, changing the string. Real entab tools usually take a `--only-leading` flag to limit replacement to indentation. Worth adding.
- The output of `detab` and `entab` should be each other's inverse for "well-behaved" input — same content, possibly different byte representation, same visual layout.
