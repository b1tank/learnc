> **Exercise 1-12.** Copy stdin to stdout but print one word per line. Define a word as any maximal run of non-whitespace characters.

## Approach

The state-machine trick from §1.5 again, but now we *emit* characters as we go instead of counting.

- If the current character is whitespace and we're inside a word, transition `IN → OUT` and emit a single `\n`.
- If it's non-whitespace, emit it and ensure we're in the `IN` state.
- If it's whitespace and we're already `OUT`, skip silently — don't emit blank lines.

```c:starter
#include <stdio.h>

int main(void) {
    /* TODO: one word per line */
    return 0;
}
```

## Solution

```c
#include <stdio.h>

#define IN  1
#define OUT 0

int main(void) {
    int c, state = OUT;

    while ((c = getchar()) != EOF) {
        if (c == ' ' || c == '\t' || c == '\n') {
            if (state == IN) {
                putchar('\n');
                state = OUT;
            }
        } else {
            putchar(c);
            state = IN;
        }
    }
    /* Optional: emit a trailing newline if the input ended inside a word */
    if (state == IN) putchar('\n');
    return 0;
}
```

## Notes from the author

- This is the same skeleton as the §1.5 counter, but the actions are I/O instead of arithmetic. That's the whole point of state-machine thinking: the *transitions* are the structure; what you do *during* the transition is interchangeable.
- The final "did we end inside a word?" check guarantees the last line is terminated. Without it, input like `foo` (no trailing newline) outputs `foo` without a newline — usually undesirable.
- The standard tool `tr -s '[:space:]' '\n'` does the same thing in one shell command. Try it side-by-side with your output.
- Bonus: replace the explicit whitespace test with `isspace(c)` from `<ctype.h>`. Adds robustness against `\r`, `\v`, `\f`, and locale-dependent whitespace.
