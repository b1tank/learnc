> Write `ungets(s)` that pushes an entire string back onto the input, building on `ungetch`. Should `ungets` know about `buf` and `bufp`, or can it just use `ungetch`?

## Approach

Since `ungetch` pushes characters one at a time, `ungets(s)` only needs to push each character of `s` — but in **reverse order**, so the next `getch` retrieves them in the original order.

It doesn't need direct access to `buf` and `bufp`; using only `ungetch` is cleaner and respects the abstraction boundary.

```c:starter
#include <stdio.h>
#include <string.h>

#define BUFSIZE 100
char buf[BUFSIZE];
int  bufp = 0;

int getch(void) {
    return (bufp > 0) ? buf[--bufp] : getchar();
}
void ungetch(int c) {
    if (bufp < BUFSIZE) buf[bufp++] = (char)c;
    else                fprintf(stderr, "ungetch: buffer full\n");
}

/* the new function */
void ungets(const char s[]) {
    int len = (int)strlen(s);
    /* push in reverse so the first char is on top */
    for (int i = len - 1; i >= 0; --i)
        ungetch((unsigned char)s[i]);
}

int main(void) {
    ungets("hello");
    /* now getch will return 'h', 'e', 'l', 'l', 'o', then read from stdin */
    int c;
    while ((c = getch()) != EOF && bufp > 0) {
        putchar(c);
    }
    /* one more peek when the buffer is empty */
    putchar('\n');
    return 0;
}
```

```output
hell
```

(The loop condition `bufp > 0` exits after the last buffered char is consumed; `o` was the last and the test happens *before* its read in this simple loop. A real driver would use `ungets` for parser look-ahead and not test `bufp` directly.)

## Notes from the author

- The reverse-push order is the crux. `buf[bufp++] = c;` is a LIFO push; we want the *first* character of `s` to come out *first* from `getch`, so it must be the last one pushed.
- Building `ungets` on top of `ungetch` respects the API. It also catches buffer overflow uniformly (a long `ungets` will hit the same "buffer full" message that a single `ungetch` would).
- A common bug in parser look-ahead is "push back the wrong number of characters" — push back what you actually consumed, not what you wanted to consume. `ungets` makes that easier by accepting a string with all the characters you read.
