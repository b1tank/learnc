> Add commands to the RPN calculator that:
>
> - **print** the top element of the stack without popping it
> - **duplicate** the top element
> - **swap** the top two elements
> - **clear** the whole stack

## Approach

Each command is a tiny operation on the stack:

| Command | Action                                                |
|---------|-------------------------------------------------------|
| `?`     | Print top                                             |
| `d`     | Push a copy of `val[sp-1]` so the top is duplicated   |
| `s`     | Swap `val[sp-1]` and `val[sp-2]`                      |
| `c`     | Reset `sp = 0`                                        |

Each command needs to guard against an under-populated stack (e.g. `swap` needs at least two elements).

```c:starter
#include <stdio.h>
#include <stdlib.h>

#define MAXVAL 100
int    sp = 0;
double val[MAXVAL];

void   push(double f) { if (sp < MAXVAL) val[sp++] = f; }
double pop(void)      { return (sp > 0) ? val[--sp] : 0.0; }

void cmd_print(void) {
    if (sp > 0) printf("top = %g\n", val[sp-1]);
    else        printf("(stack empty)\n");
}
void cmd_dup(void) {
    if (sp > 0)        push(val[sp-1]);
    else               fprintf(stderr, "dup: stack empty\n");
}
void cmd_swap(void) {
    if (sp >= 2) {
        double t = val[sp-1];
        val[sp-1] = val[sp-2];
        val[sp-2] = t;
    } else {
        fprintf(stderr, "swap: need two values\n");
    }
}
void cmd_clear(void) { sp = 0; }

int main(void) {
    push(1); push(2); push(3);
    cmd_print();                    /* top = 3 */
    cmd_dup();    cmd_print();      /* top = 3 (now stack is 1 2 3 3) */
    cmd_swap();   cmd_print();      /* top = 3 (1 2 3 3 -> 1 2 3 3 swap top two -> 1 2 3 3 ;-) */
    /* after dup+swap, stack is 1 2 3 3 ; top two same. Let's reset to demo properly: */
    cmd_clear();
    push(10); push(20);
    cmd_swap(); cmd_print();        /* top = 10 (was 20, now swapped to 10) */
    cmd_clear();
    cmd_print();                    /* (stack empty) */
    return 0;
}
```

```output
top = 3
top = 3
top = 3
top = 10
(stack empty)
```

## Notes from the author

- All four are "peek" operations on stack internals — they don't go through `push`/`pop`. This is fine; in a real codebase you'd encapsulate the stack as a struct with `peek`, `dup`, `swap` methods to keep the representation hidden.
- The "guard against insufficient stack depth" pattern is everywhere in interpreters and VMs. Every operation should know its arity and validate it.
- `swap` swaps via a temporary; the bitwise XOR swap is a fun trick (`a^=b; b^=a; a^=b;`) but it doesn't generalise to floats. Stick with the named-temp.
