> Extend the RPN calculator from §4.3 to support the modulo operator `%` (integer remainder) and to handle negative numbers cleanly.

## Approach

Two pieces:

1. **Modulo case** in the operator dispatch: pop two, compute `op2 % op1`, push back. Guard against divide-by-zero.
2. **Negative literals** in the tokenizer: a `-` not followed by a digit should be the subtraction operator; a `-` followed by a digit is the start of a negative number.

```c:starter
#include <stdio.h>
#include <stdlib.h>
#include <ctype.h>
#include <string.h>

#define MAXVAL 100

int  sp = 0;
double val[MAXVAL];

void   push(double f) { if (sp < MAXVAL) val[sp++] = f; }
double pop(void)      { return (sp > 0) ? val[--sp] : 0.0; }

/* a tiny evaluator that takes pre-split tokens */
double eval(int n_tokens, char *tokens[]) {
    sp = 0;
    for (int i = 0; i < n_tokens; ++i) {
        const char *t = tokens[i];
        /* negative literal: '-' followed by digit */
        if ((t[0] == '-' && isdigit((unsigned char)t[1])) ||
            isdigit((unsigned char)t[0])) {
            push(atof(t));
        } else if (strcmp(t, "+") == 0) {
            push(pop() + pop());
        } else if (strcmp(t, "*") == 0) {
            push(pop() * pop());
        } else if (strcmp(t, "-") == 0) {
            double op2 = pop(), op1 = pop();
            push(op1 - op2);
        } else if (strcmp(t, "/") == 0) {
            double op2 = pop(), op1 = pop();
            if (op2 != 0.0) push(op1 / op2);
            else            { fprintf(stderr, "div by zero\n"); push(0.0); }
        } else if (strcmp(t, "%") == 0) {
            int op2 = (int)pop(), op1 = (int)pop();
            if (op2 != 0) push((double)(op1 % op2));
            else          { fprintf(stderr, "mod by zero\n"); push(0.0); }
        }
    }
    return pop();
}

int main(void) {
    char *e1[] = { "23", "7", "%" };
    char *e2[] = { "-5", "3", "+" };
    char *e3[] = { "10", "-3", "*" };

    printf("23 7 %% = %g\n",  eval(3, e1));
    printf("-5 3 + = %g\n",   eval(3, e2));
    printf("10 -3 * = %g\n",  eval(3, e3));
    return 0;
}
```

```output
23 7 % = 2
-5 3 + = -2
10 -3 * = -30
```

## Notes from the author

- The "is `-` a unary sign or a binary operator?" disambiguation is a real lexer concern in any expression parser. The rule "look at the next character" works for simple cases; full languages need a state machine that tracks whether the previous token was an operator or operand.
- For modulo, we cast to `int` because C's `%` requires integer operands. To support `fmod` (float modulo), use `<math.h>`'s `fmod(a, b)`. Most RPN calculators (HP-style) actually use `fmod` so `5.5 % 1.0` makes sense.
- Note `pop() - pop()` would be a bug: the order of evaluation of the two `pop()` calls is *unspecified* in C. We explicitly grab `op2` then `op1` into named variables before subtracting.
