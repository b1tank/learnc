> Write `expr`, a command-line RPN calculator that takes its expression from `argv`. Example: `./expr 2 3 4 + \*` should print `14` (because `3 4 +` is 7, then `2 7 *` is 14).

The `*` and other shell-special characters must be escaped: `\*`, `\(`, etc., or quoted.

## Approach

Walk `argv[1..argc-1]`. For each token:

- If it parses as a number → push.
- Otherwise it's an operator → pop the right number of operands, compute, push.

When done, the top of the stack is the answer.

```c:starter
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <ctype.h>

#define MAXVAL 100
static double val[MAXVAL];
static int    sp = 0;

static void   push(double f) { if (sp < MAXVAL) val[sp++] = f; }
static double pop(void)      { return (sp > 0) ? val[--sp] : 0.0; }

static int is_number(const char *s) {
    if ((*s == '-' || *s == '+') && s[1] != '\0') ++s;
    int saw_dot = 0;
    while (*s) {
        if (*s == '.' && !saw_dot) saw_dot = 1;
        else if (!isdigit((unsigned char)*s)) return 0;
        ++s;
    }
    return 1;
}

int main(int argc, char *argv[]) {
    for (int i = 1; i < argc; ++i) {
        const char *t = argv[i];
        if (is_number(t)) {
            push(atof(t));
        } else if (strcmp(t, "+") == 0) {
            push(pop() + pop());
        } else if (strcmp(t, "*") == 0) {
            push(pop() * pop());
        } else if (strcmp(t, "-") == 0) {
            double b = pop(), a = pop();
            push(a - b);
        } else if (strcmp(t, "/") == 0) {
            double b = pop(), a = pop();
            push(b ? a / b : 0.0);
        } else {
            fprintf(stderr, "unknown token: %s\n", t);
            return 1;
        }
    }
    if (sp != 1) {
        fprintf(stderr, "stack has %d items, expected 1\n", sp);
        return 1;
    }
    printf("%g\n", pop());
    return 0;
}
```

Compile and run:

```bash
$ gcc expr.c -o expr
$ ./expr 2 3 4 + \*
14
$ ./expr 1 2 + 3 \*
9
```

## Notes from the author

- Tokens come pre-split — the shell did the whitespace work for us. That's why this exercise is much simpler than an interactive RPN calculator: no tokenisation, no whitespace handling, no line buffer.
- Shell escapes are a real concern. `*` becomes the file glob, `(` is shell grouping. Quoting (`'*'`) or escaping (`\*`) tells the shell to pass the literal character.
- A check at the end (`sp != 1`) catches user errors: too few or too many operands. Good UX for a CLI tool. This pattern — "the calculator validates its expression at the end" — is what makes a real RPN tool useful.
