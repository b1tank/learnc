> **Exercise 2-9.** On twos-complement systems, `x &= (x - 1)` clears the rightmost 1-bit of `x`. Explain why, then write a faster version of a "count the number of 1-bits in `x`" function that takes advantage of this.

## Approach

### Why `x & (x - 1)` clears the rightmost 1-bit

Suppose `x` ends in `...100…0` (one 1 followed by some zeros). Subtracting 1 turns that into `...011…1` — the rightmost 1 flips to 0, and all the zeros to its right become 1s. The bits above the cleared 1 are unaffected.

ANDing `x` with `x - 1` therefore:

- Keeps the bits above the rightmost 1 (they're 1-AND-1 = same in both)
- Clears the rightmost 1 itself (1-AND-0)
- Keeps the bits below as 0 (they were 0-AND-1 = 0)

Net effect: exactly the rightmost 1 is gone; everything else is untouched.

### Using it for a bit counter

The naïve counter loops 32 times (or however wide `x` is), checking each bit. The improved version loops **once per set bit** — much faster for sparse bit vectors.

```c:starter
#include <stdio.h>

int bitcount_slow(unsigned x);     /* naive: loop over every bit */
int bitcount_fast(unsigned x);     /* improved: loop over only the set bits */

int main(void) {
    unsigned tests[] = { 0, 1, 0xF, 0xFF, 0x55555555U, 0xFFFFFFFFU };
    for (int i = 0; i < 6; ++i)
        printf("popcount(%08X) = %d (slow), %d (fast)\n",
               tests[i], bitcount_slow(tests[i]), bitcount_fast(tests[i]));
    return 0;
}
```

## Solution

```c
int bitcount_slow(unsigned x) {
    int n = 0;
    for (; x != 0; x >>= 1)
        if (x & 1) ++n;
    return n;
}

int bitcount_fast(unsigned x) {
    int n = 0;
    while (x != 0) {
        x &= (x - 1);   /* clear the rightmost 1 */
        ++n;
    }
    return n;
}
```

```output
popcount(00000000) = 0 (slow), 0 (fast)
popcount(00000001) = 1 (slow), 1 (fast)
popcount(0000000F) = 4 (slow), 4 (fast)
popcount(000000FF) = 8 (slow), 8 (fast)
popcount(55555555) = 16 (slow), 16 (fast)
popcount(FFFFFFFF) = 32 (slow), 32 (fast)
```

The slow version always loops 32 times (until `x` becomes 0); the fast one loops only as many times as there are 1-bits. For `x = 1`, that's a 32× speedup.

## Notes from the author

- The "clear the rightmost 1" trick has more uses than just counting. To **isolate** the rightmost 1, use `x & -x` (on twos-complement). To check "is `x` a power of two?", `x && !(x & (x - 1))` — `x` is non-zero and clearing its lowest 1 leaves zero.
- For really high-performance counting, look at `__builtin_popcount(x)`. On modern x86 it compiles to a single `popcnt` instruction. The exercise version teaches the principle; the builtin shipsthe peak.
- The trick relies on twos-complement representation of negatives — specifically that subtracting 1 ripples through the trailing zeros. K&R-era C didn't *require* twos-complement, but C23 finally does.
