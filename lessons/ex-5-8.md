> The §5.7 `day_of_year` and `month_day` accept any month, any day, and any year without validation. Add error checking that rejects invalid inputs (e.g. February 30, month 13, day 0).

## Approach

Validate before computing:

- Year: positive (or your application's reasonable range, e.g. 1900–9999).
- Month: 1–12.
- Day: ≥ 1 and ≤ days-in-that-month (accounting for leap years for February).

Signal errors via a return value or out-parameter; don't crash, don't silently produce garbage.

```c:starter
#include <stdio.h>

static int is_leap(int year) {
    return (year % 4 == 0 && year % 100 != 0) || year % 400 == 0;
}

static const int day_tab[2][13] = {
    { 0, 31, 28, 31, 30, 31, 30, 31, 31, 30, 31, 30, 31 },
    { 0, 31, 29, 31, 30, 31, 30, 31, 31, 30, 31, 30, 31 },
};

/* return day-of-year, or -1 on bad input */
int day_of_year(int year, int month, int day);

/* fill *pmonth, *pday from year + day-of-year. Return 0 success, -1 bad input */
int month_day(int year, int yearday, int *pmonth, int *pday);

int main(void) {
    printf("Mar  1, 2024 = %d\n",  day_of_year(2024,  3,  1));   /* 61 */
    printf("Feb 30, 2024 = %d\n",  day_of_year(2024,  2, 30));   /* -1 */
    printf("month 13     = %d\n",  day_of_year(2024, 13,  1));   /* -1 */

    int m, d;
    int rc = month_day(2024, 60, &m, &d);
    if (rc == 0) printf("day 60 of 2024 = %d/%d\n", m, d);
    rc = month_day(2024, 1000, &m, &d);
    if (rc != 0) printf("day 1000 rejected\n");
    return 0;
}

int day_of_year(int year, int month, int day) {
    if (year < 1 || month < 1 || month > 12 || day < 1)
        return -1;
    int leap = is_leap(year);
    if (day > day_tab[leap][month])
        return -1;
    int total = day;
    for (int i = 1; i < month; ++i)
        total += day_tab[leap][i];
    return total;
}

int month_day(int year, int yearday, int *pmonth, int *pday) {
    if (year < 1 || yearday < 1 || yearday > (is_leap(year) ? 366 : 365))
        return -1;
    int leap = is_leap(year);
    int i;
    for (i = 1; yearday > day_tab[leap][i]; ++i)
        yearday -= day_tab[leap][i];
    *pmonth = i;
    *pday   = yearday;
    return 0;
}
```

```output
Mar  1, 2024 = 61
Feb 30, 2024 = -1
month 13     = -1
day 60 of 2024 = 2/29
day 1000 rejected
```

## Notes from the author

- "Validate at the boundary" is the discipline. `day_of_year` doesn't trust its inputs because they often come from users or other modules. Internal functions can skip validation if their callers already filter inputs.
- Returning `-1` as an error sentinel works here because day-of-year is always positive on valid inputs. For functions where every return value is meaningful, return errors via a separate parameter or use `errno`.
- The bound `day > day_tab[leap][month]` catches Feb 30, Apr 31, etc. — without it, the function would silently produce a wrong day-of-year by overshooting.
