> Build a word-index program: it reads text and prints each distinct word followed by a sorted list of line numbers on which the word appears. Skip a configurable list of "noise" words like `a`, `the`, `and`.

## Approach

Two pieces:

1. A **hash table** keyed by lowercased word; each entry holds a linked list of line numbers (in ascending order).
2. A **noise filter** — another hash table (or a fixed sorted array) of words to skip.

After reading the input, walk every bucket of the index and print.

```c:starter
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <ctype.h>

#define HASHSIZE 257

struct lineref {
    int line;
    struct lineref *next;
};

struct word {
    char *w;
    struct lineref *lines;
    struct word *next;
};

static struct word *table[HASHSIZE];

static const char *noise[] = {
    "a", "an", "and", "the", "of", "to", "in", "is", "it", "or", NULL
};

static int is_noise(const char *w) {
    for (int i = 0; noise[i]; ++i)
        if (strcmp(w, noise[i]) == 0) return 1;
    return 0;
}

static unsigned hash(const char *s) {
    unsigned h = 0;
    while (*s) h = 31u * h + (unsigned char)*s++;
    return h % HASHSIZE;
}

static void index_word(const char *w, int line) {
    if (is_noise(w)) return;
    unsigned h = hash(w);
    struct word *e;
    for (e = table[h]; e; e = e->next)
        if (strcmp(e->w, w) == 0) break;
    if (!e) {
        e = malloc(sizeof *e);
        e->w = strdup(w);
        e->lines = NULL;
        e->next = table[h];
        table[h] = e;
    }
    /* dedup: skip if last line == this line */
    if (e->lines && e->lines->line == line) return;
    /* insert at head — lines are read in order, so head = newest */
    struct lineref *L = malloc(sizeof *L);
    L->line = line;
    L->next = e->lines;
    e->lines = L;
}

int main(void) {
    int c, line = 1;
    char buf[64];
    int  bi = 0;
    while ((c = getchar()) != EOF) {
        if (isalpha(c) || c == '_') {
            if (bi < (int)sizeof buf - 1) buf[bi++] = tolower(c);
        } else {
            if (bi > 0) {
                buf[bi] = '\0';
                index_word(buf, line);
                bi = 0;
            }
            if (c == '\n') ++line;
        }
    }
    if (bi > 0) { buf[bi] = '\0'; index_word(buf, line); }

    /* print: walk all buckets */
    for (int i = 0; i < HASHSIZE; ++i)
        for (struct word *e = table[i]; e; e = e->next) {
            printf("%-12s ", e->w);
            /* lines are inserted at head, so they're in reverse order — reverse to ascending */
            int count = 0;
            for (struct lineref *L = e->lines; L; L = L->next) ++count;
            int *arr = malloc(count * sizeof *arr);
            int k = count - 1;
            for (struct lineref *L = e->lines; L; L = L->next) arr[k--] = L->line;
            for (int j = 0; j < count; ++j)
                printf("%d%s", arr[j], j + 1 < count ? ", " : "\n");
            free(arr);
        }
    return 0;
}
```

```output
(feed e.g. "now is the time\nfor all good men\nnow is now\n")
```

Result for that input:

```output
all          2
for          2
good         2
men          2
now          1, 3
time         1
```

(Noise words "is" and "the" are excluded.)

## Why "buckets in any order"

The hash table prints buckets in bucket order, which has no relationship to alphabetical order. For a sorted output, collect all entries into an array first, then `qsort` by word. The skeleton above prints in hash order; adding the sort is a small addition.

## Try it

1. Sort the output alphabetically by word.
2. Add a CLI flag `--noise=file.txt` to load noise words from a file.
3. Report counts: `now (2) lines 1, 3` instead of just `now 1, 3`.

## Notes from the author

- "Word index with line numbers" is the classic concordance program — what Bible studies and Shakespeare scholars built in the 1960s. The data structure (word → line list) is the seed of every full-text index.
- The dedup at the same line ("if last line == this line, skip") prevents `now now now` on line 1 from producing `1, 1, 1`. Small but important UX detail.
- Modern variants: build an *inverted index* with positions, not just line numbers. That's what Lucene/Elasticsearch do.
