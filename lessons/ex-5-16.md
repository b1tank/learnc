> Add a `-d` ("directory order") option to the sorter. In this mode, only letters, digits, and blanks count for comparison; punctuation is skipped.

## Approach

A comparator that advances past punctuation characters in both strings:

```c
int cmp_dir(const char *s, const char *t) {
    while (*s && *t) {
        while (*s && !is_directory_char((unsigned char)*s)) ++s;
        while (*t && !is_directory_char((unsigned char)*t)) ++t;
        if (!*s || !*t) break;
        if (*s != *t) return (unsigned char)*s - (unsigned char)*t;
        ++s; ++t;
    }
    return (unsigned char)*s - (unsigned char)*t;
}
```

Where `is_directory_char(c)` is `isalnum(c) || isspace(c)`.

```c:starter
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <ctype.h>

static int dir_order = 0;

static int is_dir_char(int c) {
    return isalnum(c) || isspace(c);
}

static int cmp_dir(const char *s, const char *t) {
    while (*s && *t) {
        while (*s && !is_dir_char((unsigned char)*s)) ++s;
        while (*t && !is_dir_char((unsigned char)*t)) ++t;
        if (!*s || !*t) break;
        if (*s != *t) return (unsigned char)*s - (unsigned char)*t;
        ++s; ++t;
    }
    return (unsigned char)*s - (unsigned char)*t;
}

static int cmp_lines(const void *a, const void *b) {
    const char *sa = *(const char *const *)a;
    const char *sb = *(const char *const *)b;
    return dir_order ? cmp_dir(sa, sb) : strcmp(sa, sb);
}

#define MAXLINES 5000
static char *lines[MAXLINES];

int main(int argc, char *argv[]) {
    for (int i = 1; i < argc; ++i)
        if (strcmp(argv[i], "-d") == 0) dir_order = 1;
    int n = 0;
    char buf[4096];
    while (n < MAXLINES && fgets(buf, sizeof buf, stdin)) lines[n++] = strdup(buf);
    qsort(lines, n, sizeof lines[0], cmp_lines);
    for (int i = 0; i < n; ++i) fputs(lines[i], stdout);
    return 0;
}
```

Usage:

```bash
$ printf '!apple\n!!banana\n[cherry]\n' | ./sort -d
!apple
!!banana
[cherry]
```

In directory order the punctuation `!`, `!!`, `[`, `]` is ignored, so the comparison is essentially on `apple`, `banana`, `cherry` — same alphabetical result. The difference shows up on strings that look unsorted under ASCII but sort cleanly when punctuation is excluded.

## Notes from the author

- "Directory order" is what old Unix phone-book programs used. The idea: people search by letters; the punctuation is noise. Modern locales handle this with collation rules; `strcoll` does it for you.
- The "skip non-significant chars on both sides simultaneously" loop is the general pattern for "compare with some characters elided". It generalises to "skip whitespace", "skip vowels", whatever.
- Stability matters again: two strings that are equal *under the elided comparison* may differ in punctuation. Whether they should stay in input order, or sort by the punctuation, depends on what the user wants.
