> Extend the §7.3 `minprintf` so it supports width, precision, and the `-` / `0` / `+` flags. The format `% [flags] [width] [.precision] [length] specifier` should match real `printf` for `%d`, `%f`, `%s`, `%c`.

## Approach

Parse the conversion specifier as a small state machine:

1. Flags: any of `-`, `+`, `0`, `' '`, `#`.
2. Width: optional digit sequence (or `*` to read from arg list).
3. Precision: `.` followed by digits or `*`.
4. Length modifier: `h`, `l`, `ll`, `L`, `z`.
5. Specifier: one of `d`, `u`, `f`, `e`, `g`, `s`, `c`, `x`, `o`, `%`.

Build a printf-spec string ("%6.2f") and hand it to the real `printf` / `sprintf`. This is the "compose the format from parts" approach — it works for an exercise; production-grade `printf` does its own formatting.

```c:starter
#include <stdio.h>
#include <stdarg.h>
#include <string.h>
#include <ctype.h>

static void minprintf(const char *fmt, ...) {
    va_list ap;
    va_start(ap, fmt);

    for (const char *p = fmt; *p; ) {
        if (*p != '%') { putchar(*p++); continue; }
        ++p;

        char spec[32] = "%";
        int  si = 1;

        /* flags */
        while (*p && strchr("-+ 0#", *p))
            spec[si++] = *p++;
        /* width */
        while (isdigit((unsigned char)*p))
            spec[si++] = *p++;
        /* precision */
        if (*p == '.') {
            spec[si++] = *p++;
            while (isdigit((unsigned char)*p))
                spec[si++] = *p++;
        }
        /* length modifier (skipped for brevity) */

        char conv = *p++;
        spec[si++] = conv;
        spec[si] = '\0';

        switch (conv) {
            case 'd': case 'i':
                printf(spec, va_arg(ap, int));
                break;
            case 'u': case 'x': case 'X': case 'o':
                printf(spec, va_arg(ap, unsigned));
                break;
            case 'f': case 'e': case 'g': case 'E': case 'G':
                printf(spec, va_arg(ap, double));
                break;
            case 's':
                printf(spec, va_arg(ap, const char *));
                break;
            case 'c':
                printf(spec, va_arg(ap, int));
                break;
            case '%':
                putchar('%');
                break;
            default:
                fputs(spec, stdout);
        }
    }
    va_end(ap);
}

int main(void) {
    minprintf("plain:        %d\n", 42);
    minprintf("width 6:      [%6d]\n", 42);
    minprintf("left:         [%-6d]\n", 42);
    minprintf("zero-pad:     [%06d]\n", 42);
    minprintf("plus:         [%+d]\n", 42);
    minprintf("float:        [%.2f]\n", 3.14159);
    minprintf("string clip:  [%.5s]\n", "hello world");
    minprintf("composed:     [%-10.3f]X\n", 3.14159);
    return 0;
}
```

```output
plain:        42
width 6:      [    42]
left:         [42    ]
zero-pad:     [000042]
plus:         [+42]
float:        [3.14]
string clip:  [hello]
composed:     [3.142     ]X
```

## Why "compose then call printf" works

We're effectively building the original `printf`'s spec string. Each call passes ONE argument, with its full formatting. The real `printf` does all the heavy lifting (number → digits, width padding, etc.).

This violates the spirit of "write your own", but for the *parsing* exercise it's the cleanest. A "do it all yourself" version would re-implement the number-to-string conversion — fun, but a different project.

## Try it

1. Add `*` for width and precision (read from arg list).
2. Handle the length modifiers `l` and `ll` for `long` and `long long`.
3. Replace the `printf` calls with hand-written formatters for `%d` and `%s`.

## Notes from the author

- "Build the spec and call printf" is a real-world pattern: many "extended printf" libraries (Glib's `g_string_append_printf`, etc.) parse extensions then delegate to libc's printf.
- The full conversion grammar is in C99 §7.19.6.1. It's longer than you'd think — about 60 lines of careful spec.
- For the most performance, modern printf implementations use a single state machine that walks the format string and emits formatted output directly. Skipping the "compose then re-parse" step saves microseconds per call.
