> Implement `strncpy`, `strncat`, and `strncmp`, the bounded versions of `strcpy`, `strcat`, and `strcmp`. They take a maximum count `n` to limit how many characters they touch.

## Approach

Each is a slight variant of its unbounded sibling, with a `for` loop bounded by `n`:

- **`strncpy(s, t, n)`** — copy *up to* `n` characters from `t` to `s`. If `t` is shorter, pad `s` with `'\0'`. (This is the classic gotcha — `strncpy` does NOT guarantee null termination if `t` is exactly `n` chars or longer.)
- **`strncat(s, t, n)`** — append at most `n` characters of `t` to `s`, then terminate.
- **`strncmp(s, t, n)`** — compare at most `n` characters; return as `strcmp` does.

```c:starter
#include <stdio.h>
#include <string.h>

void my_strncpy(char *s, const char *t, size_t n);
void my_strncat(char *s, const char *t, size_t n);
int  my_strncmp(const char *s, const char *t, size_t n);

int main(void) {
    char buf[16];
    my_strncpy(buf, "hello world", 5);
    buf[5] = '\0';
    printf("strncpy(buf, \"hello world\", 5) = [%s]\n", buf);

    char base[64] = "Hi, ";
    my_strncat(base, "there friend!", 5);
    printf("strncat(\"Hi, \", \"there friend!\", 5) = [%s]\n", base);

    printf("strncmp(\"hello\", \"helps\", 3) = %d\n", my_strncmp("hello", "helps", 3));
    printf("strncmp(\"hello\", \"helps\", 5) = %d\n", my_strncmp("hello", "helps", 5));
    return 0;
}

void my_strncpy(char *s, const char *t, size_t n) {
    size_t i = 0;
    for (; i < n && t[i] != '\0'; ++i)
        s[i] = t[i];
    for (; i < n; ++i)
        s[i] = '\0';
}

void my_strncat(char *s, const char *t, size_t n) {
    while (*s) ++s;
    size_t i = 0;
    while (i < n && t[i] != '\0') {
        *s++ = t[i++];
    }
    *s = '\0';
}

int my_strncmp(const char *s, const char *t, size_t n) {
    for (size_t i = 0; i < n; ++i) {
        if (s[i] != t[i])
            return (unsigned char)s[i] - (unsigned char)t[i];
        if (s[i] == '\0')
            return 0;
    }
    return 0;
}
```

```output
strncpy(buf, "hello world", 5) = [hello]
strncat("Hi, ", "there friend!", 5) = [Hi, there]
strncmp("hello", "helps", 3) = 0
strncmp("hello", "helps", 5) = -4
```

## Notes from the author

- `strncpy`'s "no null terminator if t is too long" is the most-cursed function in the standard library. Modern alternatives: `strlcpy` (BSD, glibc since 2.38), `snprintf(s, n, "%s", t)`, or your own bounded copy. The standard `strncpy` should be considered a foot-gun.
- `strncat`'s `n` is the max from `t` to append, NOT including the terminator — so the buffer needs `len(s) + n + 1` bytes. The asymmetry with `strncpy` (which counts `n` as the total written) is the reason these get confused.
- For safety, prefer `snprintf(dst, dst_sz, "%s%s", a, b)` — it bounds the *destination*, always null-terminates, and is one call instead of two. Performance is fine on modern libcs.
