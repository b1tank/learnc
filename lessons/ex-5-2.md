> Write `getfloat`, the floating-point analogue of `getint`. What type should it use to return the parsed value?

## Approach

Same shape as `getint`: skip whitespace, optional sign, integer part, optional `.` and fractional part. Return the value through a `double *` parameter, and use the integer return value to signal "next character read" or `EOF`.

`double` is the natural return type — more precision than `float`, no slower on modern hardware.

```c:starter
#include <stdio.h>
#include <ctype.h>

static int pushback;
static int has_pb = 0;

int getch(void) {
    if (has_pb) { has_pb = 0; return pushback; }
    return getchar();
}
void ungetch(int c) {
    if (has_pb) return;
    pushback = c; has_pb = 1;
}

int getfloat(double *pn);

int main(void) {
    double v;
    int rc;
    while ((rc = getfloat(&v)) != EOF) {
        if (rc == 0) printf("non-number\n");
        else         printf("got: %g\n", v);
    }
    return 0;
}

int getfloat(double *pn) {
    int c;
    int sign;

    while (isspace(c = getch()))
        ;
    if (!isdigit(c) && c != EOF && c != '+' && c != '-' && c != '.') {
        ungetch(c);
        return 0;
    }
    sign = (c == '-') ? -1 : 1;
    if (c == '+' || c == '-')
        c = getch();

    double val = 0.0;
    for (; isdigit(c); c = getch())
        val = 10.0 * val + (c - '0');

    if (c == '.') {
        double power = 1.0;
        for (c = getch(); isdigit(c); c = getch()) {
            val   = 10.0 * val + (c - '0');
            power *= 10.0;
        }
        val /= power;
    }

    *pn = sign * val;
    if (c != EOF)
        ungetch(c);
    return c;
}
```

## Notes from the author

- The integer and fractional loops are structurally identical to ex-4-2's `atof_x`. The difference is in the *driver* (read characters one at a time via `getch`) vs the *string parser* (walk an index over a buffer). Both end up with the same value.
- Skipping the exponent here keeps the code small. Add an `e`/`E` block following the same pattern from ex-4-2 to make it production-grade.
- The "`.` as legal start-of-number" detail matters: `.5` is a valid float. Without that branch, the input `.5` would be classified as "non-number".
