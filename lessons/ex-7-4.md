> Write `minscanf`, a variadic input-parser that reads `stdin` according to a format string and stores results into pointer arguments. Support `%d`, `%f`, `%s`, `%c`.

## Approach

Same structure as `minprintf`: walk the format, on each `%` pull a pointer from `va_arg`, then parse the input accordingly. Return the count of successful matches.

For each conversion:

- `%d`: skip whitespace, read optional sign + digits, convert.
- `%f`: skip whitespace, read sign + digits + `.` + digits + `e±N` if present.
- `%s`: skip whitespace, read non-whitespace into the caller's buffer.
- `%c`: read one character (no whitespace skip).

```c:starter
#include <stdio.h>
#include <stdarg.h>
#include <ctype.h>

static int read_int(int *pn) {
    int c, sign = 1;
    while (isspace(c = getchar()))
        ;
    if (c == EOF) return 0;
    if (c == '+' || c == '-') { sign = (c == '-') ? -1 : 1; c = getchar(); }
    if (!isdigit(c)) { ungetc(c, stdin); return 0; }
    int n = 0;
    do { n = 10 * n + (c - '0'); c = getchar(); } while (isdigit(c));
    if (c != EOF) ungetc(c, stdin);
    *pn = sign * n;
    return 1;
}

static int read_double(double *pn) {
    int c;
    while (isspace(c = getchar()))
        ;
    if (c == EOF) return 0;
    double sign = 1.0;
    if (c == '+' || c == '-') { sign = (c == '-') ? -1.0 : 1.0; c = getchar(); }
    if (!isdigit(c) && c != '.') { ungetc(c, stdin); return 0; }
    double v = 0;
    while (isdigit(c)) { v = 10*v + (c - '0'); c = getchar(); }
    if (c == '.') {
        double p = 1;
        c = getchar();
        while (isdigit(c)) { v = 10*v + (c-'0'); p *= 10; c = getchar(); }
        v /= p;
    }
    if (c != EOF) ungetc(c, stdin);
    *pn = sign * v;
    return 1;
}

static int read_word(char *buf, int max) {
    int c;
    while (isspace(c = getchar()))
        ;
    if (c == EOF) return 0;
    int n = 0;
    while (c != EOF && !isspace(c) && n < max - 1) {
        buf[n++] = c;
        c = getchar();
    }
    buf[n] = '\0';
    if (c != EOF) ungetc(c, stdin);
    return 1;
}

int minscanf(const char *fmt, ...) {
    va_list ap;
    va_start(ap, fmt);
    int matched = 0;

    for (const char *p = fmt; *p; ++p) {
        if (*p != '%') {
            if (isspace((unsigned char)*p)) {
                int c;
                while ((c = getchar()) != EOF && isspace(c))
                    ;
                if (c != EOF) ungetc(c, stdin);
            } else {
                int c = getchar();
                if (c != *p) { ungetc(c, stdin); break; }
            }
            continue;
        }
        ++p;
        int ok = 0;
        switch (*p) {
            case 'd': ok = read_int(va_arg(ap, int *)); break;
            case 'f': ok = read_double(va_arg(ap, double *)); break;
            case 's': ok = read_word(va_arg(ap, char *), 256); break;
            case 'c': { int c = getchar(); *(va_arg(ap, char *)) = (char)c; ok = (c != EOF); break; }
        }
        if (!ok) break;
        ++matched;
    }
    va_end(ap);
    return matched;
}

int main(void) {
    int age;
    double height;
    char name[256];
    printf("enter: name age height (e.g. 'alice 30 1.70')\n");
    int n = minscanf("%s %d %f", name, &age, &height);
    printf("parsed %d fields: %s %d %g\n", n, name, age, height);
    return 0;
}
```

```output
(awaits input — try 'alice 30 1.70')
```

## Differences from real `scanf`

- No width modifiers (so `%s` is unbounded; the real `scanf` allows `%255s`).
- No `%e`, `%g` (we treat them as `%f`).
- No conversion sets (`%[abc]`).
- Single-char ungetc only — multiple peek would need a small buffer.

These are good extensions to practise.

## Try it

1. Add `%x` for hex integers (a parser that also accepts `0x` prefix).
2. Support widths: `%32s` reads at most 32 chars.
3. Add a one-line "expect this exact text" mode (`scanf("hello %d", &x)` matches "hello " literally).

## Notes from the author

- The "two pushbacks" problem from earlier exercises bites again here. `read_int`'s lookahead at a `-` followed by a non-digit is the trickiest case; the simplified version above gets it slightly wrong (re-uses the sign for the next read).
- Real `scanf` is much more elaborate than this. Its source is 2000+ lines in glibc. The structure is the same — switch on the conversion specifier and dispatch to a type-specific parser.
- For robust input parsing in production C, use `fgets` + `strtol`/`strtod`. They give you per-field error reporting that `scanf` can't match.
