> **Exercise 2-1.** Write a program that determines the ranges of `char`, `short`, `int`, and `long` — both signed and unsigned — by printing the appropriate limits from `<limits.h>`. Then compute the same values *from the type's properties* (size and signedness) and verify the two agree.

## Approach

Two parallel computations, side by side.

1. **From `<limits.h>`**: macros like `CHAR_MIN`, `INT_MAX`, `ULONG_MAX` give the answer directly.
2. **From first principles**: for an N-bit unsigned type, the max is $2^N - 1$. For an N-bit twos-complement signed type, the max is $2^{N-1} - 1$ and the min is $-2^{N-1}$.

Use `sizeof` × `CHAR_BIT` to get the bit width without hard-coding.

```c:starter
#include <stdio.h>
#include <limits.h>

int main(void) {
    /* TODO: print both library-reported and computed ranges */
    return 0;
}
```

## Solution

```c
#include <stdio.h>
#include <limits.h>

int main(void) {
    int bits_per_byte = CHAR_BIT;
    printf("CHAR_BIT = %d\n\n", bits_per_byte);

    printf("%-8s %-6s %-22s %-22s\n",
           "type", "bytes", "from <limits.h>", "from sizeof");

    /* signed char */
    printf("%-8s %-6zu [%d, %d]   ",
           "char", sizeof(char), SCHAR_MIN, SCHAR_MAX);
    long lo = -(1L << (sizeof(char) * bits_per_byte - 1));
    long hi =  (1L << (sizeof(char) * bits_per_byte - 1)) - 1;
    printf("[%ld, %ld]\n", lo, hi);

    /* unsigned char */
    printf("%-8s %-6zu [0, %u]   ",
           "uchar", sizeof(unsigned char), UCHAR_MAX);
    unsigned long uhi = (1UL << (sizeof(unsigned char) * bits_per_byte)) - 1;
    printf("[0, %lu]\n", uhi);

    /* short */
    printf("%-8s %-6zu [%d, %d] ",
           "short", sizeof(short), SHRT_MIN, SHRT_MAX);
    lo = -(1L << (sizeof(short) * bits_per_byte - 1));
    hi =  (1L << (sizeof(short) * bits_per_byte - 1)) - 1;
    printf("[%ld, %ld]\n", lo, hi);

    /* int */
    printf("%-8s %-6zu [%d, %d]   ",
           "int", sizeof(int), INT_MIN, INT_MAX);
    /* sizeof(int)*CHAR_BIT - 1 may equal sizeof(long)*CHAR_BIT - 1; that's
       why we shift by (n-1) and double-then-subtract, not 1<<n directly */
    long imax = (1L << (sizeof(int) * bits_per_byte - 1)) - 1;
    printf("[%ld, %ld]\n", -imax - 1, imax);

    /* long needs %ld */
    printf("%-8s %-6zu [%ld, %ld]\n",
           "long", sizeof(long), LONG_MIN, LONG_MAX);

    return 0;
}
```

```output
CHAR_BIT = 8

type     bytes  from <limits.h>        from sizeof
char     1      [-128, 127]   [-128, 127]
uchar    1      [0, 255]   [0, 255]
short    2      [-32768, 32767] [-32768, 32767]
int      4      [-2147483648, 2147483647]   [-2147483648, 2147483647]
long     8      [-9223372036854775808, 9223372036854775807]
```

(Numbers will differ on a 32-bit machine — that's the whole point of computing them from `sizeof`.)

## Notes from the author

- The trick is `1L << (sizeof(T) * CHAR_BIT - 1)`. The `L` matters: on a 32-bit platform, `1 << 31` is undefined behaviour for plain `int` because it overflows the sign bit. `1L` (or `1UL`) sidesteps that.
- C does not technically *require* twos-complement until C23. Before that, `INT_MIN` could in principle be `-INT_MAX` (signed-magnitude) or `-INT_MAX-1` (twos-complement). In practice every hosted platform you'll meet is twos-complement. C23 finally bakes that in.
- `CHAR_BIT` is almost always `8` on hosted platforms but can be larger on some DSP chips. Using it (rather than hard-coded 8) is the portable habit.
