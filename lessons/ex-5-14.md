> Modify the line-sorting program from §5.6 to accept a `-r` flag that sorts in reverse (decreasing) order.

## Approach

A boolean flag toggles the comparator. The comparator can either negate its result, or you can have two named comparators and pick one.

```c:starter
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

static int reverse = 0;

static int cmp_lines(const void *a, const void *b) {
    const char *sa = *(const char *const *)a;
    const char *sb = *(const char *const *)b;
    int r = strcmp(sa, sb);
    return reverse ? -r : r;
}

#define MAXLINES 5000
static char *lines[MAXLINES];

int main(int argc, char *argv[]) {
    for (int i = 1; i < argc; ++i)
        if (strcmp(argv[i], "-r") == 0) reverse = 1;

    int n = 0;
    char buf[4096];
    while (n < MAXLINES && fgets(buf, sizeof buf, stdin)) {
        lines[n++] = strdup(buf);
    }
    qsort(lines, n, sizeof lines[0], cmp_lines);
    for (int i = 0; i < n; ++i) fputs(lines[i], stdout);
    for (int i = 0; i < n; ++i) free(lines[i]);
    return 0;
}
```

Usage:

```bash
$ printf 'banana\napple\ncherry\n' | ./sort
apple
banana
cherry
$ printf 'banana\napple\ncherry\n' | ./sort -r
cherry
banana
apple
```

## Notes from the author

- "Pass a context flag into the comparator" works because `qsort`'s comparator is called through a function pointer that has access to file-scope globals. The cleaner version is `qsort_r` (GNU extension) which passes a context pointer to the comparator — no globals needed.
- Negating the result is fine for `strcmp`, but be careful: `INT_MIN` negated is undefined behaviour. `strcmp` returns "negative, zero, or positive" — not specifically `-1`/`+1` — so theoretically a comparator returning `INT_MIN` could break. In practice the values are tiny.
- Each sort option (`-r`, `-f`, `-d`, ...) is one flag and one tweak to the comparator. The structure scales as long as the comparator stays comparison-based. For multi-key sort, the comparator becomes a small state machine.
