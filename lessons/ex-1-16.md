> **Exercise 1-16.** Revise the longest-line program from §1.9 so it correctly reports the length of *arbitrarily long* input lines, even when they exceed `MAXLINE`, and saves as much of the longest line as the buffer can hold.

## Approach

The §1.9 program had two limitations:

1. **Length reporting was wrong for long lines.** `read_line` stopped reading at `MAXLINE - 1` characters and reported *that* count, not the real line length.
2. **The saved copy got truncated silently.** No flag, no indicator.

The fix: keep reading characters even after the buffer is full. Just stop *storing* them — keep counting until you hit `\n` or `EOF`. That way `len` is the real length, and the saved buffer holds whatever fits.

```c:starter
#include <stdio.h>

#define MAXLINE 10   /* tiny on purpose, so long lines are common */

int  read_line(char line[], int maxsize);
void copy_line(char dest[], const char src[]);

int main(void) {
    /* TODO */
    return 0;
}
```

## Solution

```c
#include <stdio.h>

#define MAXLINE 10

int  read_line(char line[], int maxsize);
void copy_line(char dest[], const char src[]);

int main(void) {
    char line[MAXLINE];
    char longest[MAXLINE];
    int  len, max = 0;

    while ((len = read_line(line, MAXLINE)) > 0) {
        if (len > max) {
            max = len;
            copy_line(longest, line);
        }
    }

    if (max > 0)
        printf("longest line: %d chars (saved %d)\n%s%s",
               max,
               (max < MAXLINE) ? max : MAXLINE - 1,
               longest,
               (max >= MAXLINE) ? " [truncated]\n" : "");

    return 0;
}

/* read_line: read a logical line; return its TRUE length (excluding \n).
   At most maxsize-1 characters are stored in s, then '\0'.
   Continues consuming chars beyond the buffer so len reflects reality. */
int read_line(char s[], int maxsize) {
    int c, i = 0, len = 0;

    while ((c = getchar()) != EOF && c != '\n') {
        ++len;
        if (i < maxsize - 1) {
            s[i++] = c;
        }
        /* otherwise: discard but keep counting */
    }
    if (c == '\n') {
        ++len;
        if (i < maxsize - 1) s[i++] = '\n';
    }
    s[i] = '\0';
    return len;
}

void copy_line(char dest[], const char src[]) {
    int i = 0;
    while ((dest[i] = src[i]) != '\0')
        ++i;
}
```

## Notes from the author

- This is the first exercise that hints at why "always carry sizes" is a virtue. The fix is small but the new contract — *len is the real length, stored bytes may be fewer* — has to be communicated in comments because the function signature can't express it.
- A more idiomatic modern fix would be to `realloc` the buffer when it fills. That's chapter 7 territory; in the K&R-only world, "truncate and warn" is the move.
- The reporting message is deliberately frank about the truncation. Silent truncation in log-processing pipelines is the cause of more than its share of confused engineers; better to print a `[truncated]` marker.
- Stretch: change `MAXLINE` to a runtime parameter passed in from the command line. That's a clean §5.10 follow-up.
