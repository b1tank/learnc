> Write `strrindex(s, t)` that returns the **rightmost** position in `s` where `t` occurs, or `-1` if `t` does not occur. (Compare with the `strindex` in §4.1, which returns the leftmost.)

## Approach

Two natural strategies:

1. **Scan once from left to right**, remembering the most recent match. After the scan, that's the rightmost.
2. **Scan from right to left**, return the first match.

Strategy 1 visits every character once; strategy 2 may return early but the loop body is messier. We'll use strategy 1.

```c:starter
#include <stdio.h>

int strrindex(const char s[], const char t[]);

int main(void) {
    printf("%d\n", strrindex("the quick brown fox jumps over the lazy dog", "the"));   /* expect 31 */
    printf("%d\n", strrindex("the the the",                                   "the"));   /* expect 8  */
    printf("%d\n", strrindex("hello",                                         "world")); /* expect -1 */
    printf("%d\n", strrindex("aaaa",                                          "aa"));    /* expect 2  */
    return 0;
}
```

## Solution

```c
int strrindex(const char s[], const char t[]) {
    int found = -1;

    for (int i = 0; s[i] != '\0'; ++i) {
        int j, k;
        for (j = i, k = 0; t[k] != '\0' && s[j] == t[k]; ++j, ++k)
            ;
        if (k > 0 && t[k] == '\0')
            found = i;    /* keep updating; we want the LAST match */
    }
    return found;
}
```

```output
31
8
-1
2
```

## Notes from the author

- For tiny strings, the linear scan from the left works fine. For very long `s` with `|t|` small, scanning from the right is `O(|s|)` worst case as well but tends to terminate sooner on average.
- The empty-needle case (`t == ""`) is a classic edge: should it return `strlen(s)` (rightmost insert position) or `0`? Our implementation returns `-1` (because `k > 0` rejects zero-length matches). Many standard `strrstr` implementations return the *end* of the string. Pick a convention and document it.
- The `k > 0` guard is what handles the empty-`t` case. Without it, every position would "match" and you'd return `strlen(s)-1`.
