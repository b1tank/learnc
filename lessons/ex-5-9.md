> Rewrite `day_of_year` and `month_day` so they use pointer arithmetic to access `day_tab` instead of subscripting.

## Approach

`day_tab[leap][i]` is `*(*(day_tab + leap) + i)`. We can walk a pointer into the row:

```c
const int *row = day_tab[leap];   /* pointer to the leap-aware row */
for (int i = 1; i < month; ++i)
    total += row[i];              /* still subscripts row, but row is a pointer */
```

For pure pointer arithmetic with no subscripts:

```c
const int *row = day_tab[leap] + 1;     /* skip the leading 0 */
const int *end = row + (month - 1);     /* one past the months we want */
while (row < end)
    total += *row++;
```

```c:starter
#include <stdio.h>

static int is_leap(int year) {
    return (year % 4 == 0 && year % 100 != 0) || year % 400 == 0;
}

static const int day_tab[2][13] = {
    { 0, 31, 28, 31, 30, 31, 30, 31, 31, 30, 31, 30, 31 },
    { 0, 31, 29, 31, 30, 31, 30, 31, 31, 30, 31, 30, 31 },
};

int day_of_year_p(int year, int month, int day) {
    if (year < 1 || month < 1 || month > 12 || day < 1)
        return -1;
    const int *p = day_tab[is_leap(year)];   /* points to row */
    if (day > p[month]) return -1;
    int total = day;
    const int *q   = p + 1;                  /* first real month entry */
    const int *end = p + month;              /* exclusive */
    while (q < end)
        total += *q++;
    return total;
}

int main(void) {
    printf("Mar  1, 2024 = %d\n", day_of_year_p(2024,  3,  1));
    printf("Dec 31, 2024 = %d\n", day_of_year_p(2024, 12, 31));
    return 0;
}
```

```output
Mar  1, 2024 = 61
Dec 31, 2024 = 366
```

## Notes from the author

- `day_tab[is_leap(year)]` decays to a pointer to the appropriate row. The expression "an array's name as an rvalue is a pointer to the first element" applies to the inner row too.
- The pointer-walk version produces the same `O(month)` loop. Modern compilers vectorise or unroll both forms identically.
- This is the exact spot where pointer style is *not* better than subscripts. `p[month]` and `p + 1` are both pointer expressions; the explicit `*q++` adds noise without adding clarity. Use whichever reads better.
