> **Exercise 2-7.** Write `invert(x, p, n)` that returns `x` with the `n` bits starting at position `p` inverted (1 becomes 0, 0 becomes 1). All other bits of `x` are unchanged.

## Approach

XOR is the bit-flip operator: `b ^ 1 = !b`, `b ^ 0 = b`. So building a mask with 1s exactly where we want to flip and XOR-ing it with `x` does the job.

```c:starter
#include <stdio.h>

unsigned invert(unsigned x, int p, int n);

int main(void) {
    /* Flip bits 4..7 of 0xFF00. Result: 0xF000 (since those bits were 0). */
    printf("%X\n", invert(0xFF00, 4, 4));    /* expect F0F0 */
    /* Flip bits 0..3 of 0xFFFF. Bits 0..3 go from 1111 to 0000. */
    printf("%X\n", invert(0xFFFF, 0, 4));    /* expect FFF0 */
    return 0;
}

unsigned invert(unsigned x, int p, int n) {
    /* TODO */
    return x;
}
```

## Solution

```c
unsigned invert(unsigned x, int p, int n) {
    unsigned mask = ((1U << n) - 1) << p;    /* n ones at position p */
    return x ^ mask;
}
```

```output
F0F0
FFF0
```

## Notes from the author

- The XOR-toggle is one of the most elegant uses of the bitwise operators. You don't need to know what the original bits are — XOR with the same mask flips each bit exactly once.
- This is also the foundation of one-time-pad encryption and many parity/checksum schemes. The double-XOR identity `(a ^ b) ^ b == a` lets you undo a transformation by reapplying it.
- Like `setbits` in the previous exercise, this routine has the same `n == width` edge case. In production code, replace `(1U << n) - 1` with `n >= width ? ~0U : (1U << n) - 1` to handle the whole word.
