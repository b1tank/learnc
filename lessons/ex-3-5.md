> Write `itob(n, s, b)` that converts integer `n` into a base-`b` string in `s`. Use digits `0-9` and letters `a-z` (or `A-Z`) for bases up to 36. For example `itob(255, s, 16)` should produce `"ff"`.

## Approach

Same structure as `itoa`, but the digit table is wider:

- 0–9 for digit values 0–9 → `digit + '0'`
- 10–35 for digit values a–z → `digit - 10 + 'a'`

A small character lookup table is the cleanest way:

```c
static const char digits[] = "0123456789abcdefghijklmnopqrstuvwxyz";
s[i++] = digits[n % b];
```

```c:starter
#include <stdio.h>
#include <string.h>
#include <limits.h>

void itob(int n, char s[], int b);
void reverse(char s[]);

int main(void) {
    char buf[64];

    itob(255,    buf, 16); printf("255 base 16 = %s\n", buf);
    itob(255,    buf, 2);  printf("255 base 2  = %s\n", buf);
    itob(1234,   buf, 36); printf("1234 base 36 = %s\n", buf);
    itob(-42,    buf, 16); printf("-42 base 16 = %s\n", buf);
    itob(0,      buf, 10); printf("0 base 10   = %s\n", buf);
    itob(INT_MIN, buf, 16); printf("INT_MIN base 16 = %s\n", buf);
    return 0;
}
```

## Solution

```c
void itob(int n, char s[], int b) {
    static const char digits[] = "0123456789abcdefghijklmnopqrstuvwxyz";
    int sign = n;
    int i = 0;

    if (b < 2 || b > 36) {        /* defensive: out of range */
        s[0] = '\0';
        return;
    }

    do {
        int d = n % b;
        if (d < 0) d = -d;        /* INT_MIN safety, as in ex-3-4 */
        s[i++] = digits[d];
        n /= b;
    } while (n != 0);

    if (sign < 0)
        s[i++] = '-';

    s[i] = '\0';
    reverse(s);
}

void reverse(char s[]) {
    int i = 0, j = (int)strlen(s) - 1;
    while (i < j) {
        char t = s[i]; s[i] = s[j]; s[j] = t;
        ++i; --j;
    }
}
```

```output
255 base 16 = ff
255 base 2  = 11111111
1234 base 36 = ya
-42 base 16 = -2a
0 base 10   = 0
INT_MIN base 16 = -80000000
```

## Notes from the author

- The digit table `"0123456789abcdef...xyz"` is a *constant lookup* that's far cleaner than nested conditionals for "is this digit 0-9 or a-z". This is a classic table-driven trick: replace conditional logic with array indexing.
- Bounds-checking `b` (must be between 2 and 36) is the kind of "validate at the boundary" rule from the project's implementation discipline. The function can't do anything useful with `b == 0` or `b == 100`; failing early prevents weird output.
- The "negative-form arithmetic + flip on output" trick from ex-3-4 carries over verbatim. Once you write it once, you reach for it whenever you do integer-to-string in any base.
